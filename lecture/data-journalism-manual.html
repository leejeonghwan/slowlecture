파이썬이란 무엇인가?

파이썬(Python)은 1991년 귀도 반 로섬(Guido van Rossum)이 개발한 고급 프로그래밍 언어입니다. 영어와 비슷한 문법으로 쉽게 배울 수 있으면서도 매우 강력한 기능을 제공합니다.

왜 저널리스트는 파이썬을 배워야 할까?

  • 빠른 분석: 수천 개의 데이터를 몇 초 안에 분석
  • 재현 가능성: 같은 분석을 언제든 반복할 수 있음
  • 투명성: 독자들에게 분석 과정을 보여줄 수 있음
  • 확장성: 데이터 규모가 커져도 같은 코드 사용 가능
  • 자동화: 반복되는 작업을 자동화하여 시간 절약

파이썬 코드 실행 환경.

파이썬 코드를 실행하는 여러 가지 방법이 있습니다. 상황에 맞는 환경을 선택할 수 있습니다.

🏠 부동산 데이터 스크래핑 및 분석 ▼

호텔링 시스템 등의 부동산 사이트에서 매물 정보를 수집하고, 가격 추이를 추적할 수 있습니다. 특정 지역의 시세 변동을 자동으로 모니터링하고 기사 아이디어를 발굴할 수 있습니다.

# 웹사이트에서 부동산 정보 추출
from bs4 import BeautifulSoup

response = requests.get(url)
soup = BeautifulSoup(response.content, 'html.parser')

# 가격 데이터 추출

for property_elem in soup.find_all('div', class_='property'):

    prices.append(price)
🗳️ 선거 데이터 분석 및 시각화 ▼

선거관리위원회에서 제공하는 개표 데이터를 분석하여 지역별 투표 경향, 당선자 변화 추이 등을 추출할 수 있습니다. 판도 변화를 실시간으로 추적하는 대시보드를 만들 수도 있습니다.

개념적 예제 
# 선거 데이터 읽기 및 분석
import pandas as pd

# CSV 파일에서 데이터 로드
election_data = pd.read_csv('election_results.csv')

# 지역별 평균 득표율 계산
by_region = election_data.groupby('지역')['득표율'].mean()

closest_races = election_data.nsmallest(5, '당선자_차이')

파이썬의 강점과 약점.

✅ 파이썬이 좋은 경우.

  • 데이터 분석 및 통계 처리.
  • 웹 스크래핑 및 API 통신.
  • CSV, JSON, XML 등 파일 처리.
  • 반복되는 작업의 자동화.
  • 머신러닝 및 텍스트 분석.

❌ 다른 도구가 나은 경우.

  • 빠른 시각화: 엑셀, 태블로(Tableau)
  • 간단한 계산: 계산기, 엑셀
  • 웹 애플리케이션: JavaScript, React
  • 시스템 프로그래밍: C, Rust

주의: 파이썬은 은탄환이 아닙니다.

모든 작업에 파이썬이 최선인 것은 아닙니다. 상황에 맞는 도구를 선택하는 것이 중요합니다. 간단한 데이터라면 엑셀로 충분할 수 있습니다.

변수와 자료형의 기초.

파이썬에서는 데이터를 저장하기 위해 변수를 사용합니다. 각 변수는 정해진 자료형을 가지고 있습니다.

자료형 (Data Types)

가장 기본이 되는 세 가지 자료형을 알아봅시다:

int (정수) - 숫자 전체 ▼

음수, 0, 양수 등 소수점이 없는 숫자입니다.

Python 
# 뉴스룸 예제: 기사 댓글 수
article_comments = 2547  # 정수
published_year = 2024

# 음수도 가능
temperature_change = -3  # 기후 데이터

print(article_comments)
print(type(article_comments))

Output: 2547
<class 'int'>

float (실수) - 소수점이 있는 숫자 ▼

소수점이 있는 숫자들입니다. 평균, 백분율, 측정 단위 등에 자주 사용됩니다.

Python 
# 뉴스룸 예제: 통계 데이터
unemployment_rate = 3.5  # 실수
average_salary = 45620000.50  # 평균 급여
approval_rating = 0.487  # 지지율 48.7%

print(unemployment_rate)
print(type(approval_rating))

Output: 3.5
<class 'float'>

str (문자열) - 텍스트 ▼

따옴표로 감싼 텍스트입니다. 기사 제목, 저자 이름 등 텍스트 정보를 저장합니다.

변수 (Variables)

변수는 데이터를 저장하는 상자라고 생각하면 됩니다. 변수에 이름을 붙이고 값을 할당합니다.

변수 할당 예제 
# 변수에 값 할당
시도명 = "서울"
인구수 = 9776000

# 변수 사용
print(시도명)
print(f"{시도명}의 인구: {인구수:,}")

변수 이름 규칙.

  • 한글, 영문, 숫자, 언더스코어(_) 사용 가능.
  • 숫자로 시작할 수 없음 (예: 2024year 불가)
  • 띄어쓰기 불가 (대신 언더스코어 사용: my_variable)
  • 대소문자 구분 (Name과 name은 다른 변수)
  • 파이썬 예약어는 사용 불가 (if, for, while 등)
  • 권장: 의미 있는 이름 사용 (n보다는 article_count)
좋은 변수 이름 vs 나쁜 이름

리스트 (Lists)

여러 개의 데이터를 순서대로 저장하는 자료구조입니다. 대괄호 []로 표시합니다.

리스트 생성 및 접근 
# 리스트 생성
광역시도 = ["서울", "부산", "대구", "인천", "광주"]

# 첫 번째 항목 접근 (0부터 시작)
print(광역시도[0])  # "서울" 출력

# 마지막 항목
print(광역시도[-1])  # "광주" 출력

# 리스트 길이
print(len(광역시도))  # 5 출력
리스트 항목 추가 및 수정 
# 인구수 리스트

# 항목 수정

# 항목 제거

print(인구수)

리스트 인덱싱 (Indexing)

파이썬에서는 0부터 시작하는 인덱스를 사용합니다:

  • list[0] = 첫 번째 항목
  • list[1] = 두 번째 항목
  • list[-1] = 마지막 항목
  • list[-2] = 마지막에서 두 번째

함수란?

함수(Function)는 특정 작업을 수행하도록 작성된 코드 블록입니다. 같은 작업을 반복할 때 함수를 정의하면 코드를 재사용할 수 있고 가독성이 높아집니다.

함수 사용의 장점.

  • 재사용: 같은 코드를 여러 번 작성할 필요 없음
  • 가독성: 코드의 의도가 명확해짐
  • 유지보수: 수정 시 한 곳만 변경하면 됨
  • 테스트: 각 함수를 독립적으로 테스트 가능
  • 협업: 팀 멤버가 쉽게 이해하고 사용 가능

def election_analysis(votes_a, votes_b, total):

객체지향 프로그래밍이란?

객체지향 프로그래밍(OOP, Object-Oriented Programming)은 현실의 물건을 모델링하는 프로그래밍 방식입니다. 데이터와 그 데이터를 처리하는 함수를 함께 묶습니다.

  • 메서드: 객체가 할 수 있는 동작 (예: 기사 저장, 발행)

클래스(Class)는 객체의 설계도입니다. 클래스를 정의하고 그로부터 객체(인스턴스)를 만들 수 있습니다.

# 클래스 정의
class Article:
    """뉴스 기사를 나타내는 클래스"""

    # 생성자 (객체 초기화)
    def __init__(self, title, author):
        # 속성 설정
        self.title = title
        self.author = author
        self.views = 0
        self.published = False

    # 메서드 (동작)

        """기사 발행"""
        self.published = True
        print(f"기사 발행: {self.title}")

    def add_view(self):
        """조회수 증가"""
        self.views += 1

    def info(self):
        """기사 정보 출력"""
        return f"{self.title} (작성자: {self.author}, 조회: {self.views})"

# 객체 생성 (인스턴스화)
article1 = Article("정부 정책 발표", "김철수")

# 메서드 호출

article1.add_view()
article1.add_view()

print(article1.info())
print(article2.info())

        self.name = name
        self.party = party
        self.district = district
        self.votes = 0
        self.campaign_promises = []

    def add_votes(self, count):

        self.votes += count

        """공약 추가"""

        if total_votes == 0:

        return (self.votes / total_votes) * 100

import requests

def safe_api_call(url, params=None, max_retries=3):
    """안전한 API 호출 함수"""
    for attempt in range(max_retries):
        try:
            response = requests.get(url, params=params, timeout=10)
            response.raise_for_status()  # 에러 상태 코드 시 예외 발생
            return response.json()

        except requests.exceptions.Timeout:
            print(f"타임아웃 ({attempt+1}/{max_retries})")
        except requests.exceptions.ConnectionError:
            print(f"연결 오류 ({attempt+1}/{max_retries})")
        except requests.exceptions.HTTPError as e:
            print(f"HTTP 오류: {e.response.status_code}")
            return None

    print("재시도 횟수 초과")
    return None

# 사용 예
data = safe_api_call('https://api.example.com/data', {'year': 2025})

M14. 웹 스크래핑이란.

초급 20분

웹 스크래핑의 정의와 저널리즘에서의 가치.

웹 스크래핑(Web Scraping)은 웹사이트에서 자동으로 정보를 수집하는 기술입니다. 브라우저에서 수동으로 복사-붙여넣기를 하는 대신, 프로그램이 대량의 데이터를 체계적으로 모아줍니다.

왜 기자에게 필요한가?
  • 공개 데이터가 산재해 있을 때 한곳으로 모으기.
  • 시간이 지나면서 변하는 데이터의 변화 추적
  • 대량 데이터(수천 건 이상)를 손으로 수집할 수 없을 때.
  • 정부/기관이 다운로드 옵션을 제공하지 않을 때.
  • 독점 데이터 확보로 뉴스의 차별성 확보.

해외 주요 사례.

2020년 미국 경찰의 총기 사용으로 인한 사망자를 추적했습니다. 경찰청 웹사이트, 언론 기사, 공개 기록 등 산재된 정보를 모아 전국 규모의 데이터베이스를 구축했습니다.

미국 병원들의 의료비 청구 기준을 웹에서 수집하여 비교 분석했습니다. 각 병원 웹사이트에 산재된 가격 데이터를 모아 대중이 접근 가능한 데이터베이스로 재가공했습니다.

한국의 주요 사례.

뉴스타파
  • 국회의원 재산 변동, 퇴직금 추적.
  • 공공기관 채용공고에서 특정 지역 편향 분석.
  • 법원 판결문 데이터베이스 수집.
한겨레 데이터저널리즘팀
  • 부동산 실거래가 분석 (부동산원, 국토교통부 데이터)
  • 은행 대출금리 변동 추적.
  • 대학 등록금 변동 패턴 분석.

법적, 윤리적 고려사항 (한국)

한국의 관련 법률.

법률 내용 스크래핑 영향
저작권법 원저작물 보호 텍스트, 이미지 등 저작권 있는 내용 수집 시 주의
정보통신망법 불법적 접근 금지 공개 페이지는 OK, 인증 우회는 불법
개인정보보호법 개인정보 수집·이용 제한 이름, 전화번호, 주소 등 개인정보 수집 시 주의

핵심 원칙: 공개된 데이터, 적절한 목적, 개인정보 미수집이면 대부분 안전합니다.

기자의 윤리 기준.

  • 투명성: 데이터 수집 방식을 기사에 명시
  • 목적성: 공익을 위한 명확한 보도 목적
  • 정확성: 수집한 데이터 검증
  • 존중: 웹사이트 운영자의 서버 부하 배려 (속도 조절)
  • 출처표시: 데이터 출처 명시

4

포함되면 개인정보보호법 확인 필수입니다.

기자에게 물어보기

법무팀이 있는 언론사면 검토를 요청합니다.

언제 스크래핑하면 안 되는가?

피해야 할 상황:
  • API가 있는 경우: 공식 API를 제공하는 웹사이트는 스크래핑보다 API 사용
  • 로그인 필수: 개인 계정 없이는 접근 불가한 데이터는 수집 금지
  • robots.txt 거부: 명시적으로 자동 수집을 금지하는 사이트
  • 과도한 접근: 서버에 부하를 주는 속도로 대량 수집
  • 지적재산권 침해: 저작물(글, 사진, 영상) 무단 배포

대신 사용할 수 있는 방법들.

방법 장점 단점
공식 API 안전하고 빠름 제공 사이트가 제한적
데이터 다운로드 합법적, 정확함 모든 사이트가 제공하지 않음
정보공개청구 정부 데이터는 반드시 공개 시간 소요 (2~3주)
공개 데이터 포털 이미 정제된 데이터 원하는 형식이 없을 수 있음
언론사 제휴 데이터 접근 가능 상용 서비스

M16. 스크래핑 기초: requests + BeautifulSoup.

중급 45분

requests 라이브러리: 웹페이지 가져오기.

requests는 웹페이지의 HTML 코드를 다운로드하는 라이브러리입니다. 브라우저를 열지 않고도 Python에서 웹페이지를 "요청"할 수 있습니다.

requests 설치 및 기본 사용법 
# 설치
# pip install requests

import requests

# 웹페이지 요청
url = "https://example.com/news"
response = requests.get(url)

# 상태 확인 (200 = 성공)
print(response.status_code)  # 200

# HTML 코드 보기
html = response.text
print(html[:500])  # 처음 500자
response.status_code 의미:
  • 200: 성공.
  • 404: 페이지 없음.
  • 403: 접근 거부.
  • 500: 서버 오류.

BeautifulSoup: HTML 파싱.

BeautifulSoup은 HTML 코드를 읽기 좋게 정리해주는 라이브러리입니다. 마치 문서 목차를 이용해 특정 섹션을 찾는 것처럼, HTML에서 원하는 데이터를 추출할 수 있습니다.

BeautifulSoup 설치 및 기본 사용법 
# 설치
# pip install beautifulsoup4

from bs4 import BeautifulSoup
import requests

# 페이지 가져오기
url = "https://example.com/news"
response = requests.get(url)
html = response.text

# BeautifulSoup으로 파싱
soup = BeautifulSoup(html, 'html.parser')

# 첫 번째 제목 찾기
title = soup.find('h1')
print(title.text)  # 텍스트만 추출

주요 메서드: find()와 find_all()

find() - 첫 번째 매칭 요소 찾기.

find() 사용 예제 
# 특정 태그 찾기
h1 = soup.find('h1')

# class로 찾기
article = soup.find('div', class_='article')

# id로 찾기
main = soup.find('div', id='main-content')

# 속성으로 찾기
link = soup.find('a', attrs={'data-id': '123'})

# 텍스트 추출
if h1:
    text = h1.text  # 또는 h1.get_text()
    print(text)

requests로 안 되는 이유: JavaScript 렌더링.

일부 웹사이트(특히 최신 프레임워크로 만든 사이트)는 JavaScript를 이용해 브라우저에서 콘텐츠를 동적으로 생성합니다. 이런 경우 requests.get()으로는 빈 HTML만 받아집니다.

JavaScript 사이트 특징:
  • 페이지 소스(Ctrl+U)에는 데이터가 거의 없음.
  • 개발자 도구 Elements에는 데이터가 있음 (js가 생성함)
  • Network 탭에서 JSON API 응답 확인 가능.
  • 페이지를 내릴수록 더 많은 콘텐츠가 로드됨 (무한 스크롤)

해결법 1: 숨겨진 API 찾기 (Skip Scraping)

"Skip Scraping"은 HTML을 파싱하지 않고 웹사이트가 내부적으로 사용하는 API를 직접 호출하는 기법입니다. 이것이 가장 효율적이고 안정적입니다.

1
개발자 도구 → Network 탭 열기
2
웹페이지 새로고침: F5 또는 Ctrl+R
3
요청 목록 확인: XHR (XMLHttpRequest) 필터링
4
JSON 응답 찾기: Response 탭에서 JSON 형식의 데이터 확인
5
요청 URL 복사: Headers 탭에서 Request URL 복사
API를 직접 호출한 예제 
import requests
import json

# Network 탭에서 찾은 API URL
api_url = "https://example.com/api/v1/news"

# 필요한 헤더 설정
headers = {
    'User-Agent': 'Mozilla/5.0...',
    'Accept': 'application/json'
}

# 쿼리 파라미터
params = {
    'page': 1,
    'limit': 20,
    'sort': 'latest'
}

# API 호출
response = requests.get(api_url, headers=headers, params=params)
data = response.json()  # JSON으로 파싱

# 데이터 처리
for item in data.get('articles', []):
    title = item['title']

  • 데이터 신뢰성 높음.
  • 페이지 변경의 영향 적음.

melt() — Wide를 Long으로.

# Wide 형식 DataFrame
df_wide = pd.DataFrame({
    '시도': ['서울', '부산', '대구'],
    2021: [9400000, 3340000, 2370000],
    2022: [9406000, 3345000, 2378000],
    2023: [9411000, 3350000, 2385000]
})

# Long 형식으로 변환
df_long = df_wide.melt(
    id_vars=['시도'],  # 변하지 않는 열
    var_name='연도',  # 이전 열 이름들이 들어갈 새 열
    value_name='인구'  # 값들이 들어갈 새 열
)

print(df_long)

pivot_table() — Long을 Wide로.

Long 형식 데이터를 다시 Wide로 돌릴 때, 그리고 더 복잡한 집계를 할 때 사용합니다.

# Long 형식을 Wide로
df_pivot = df_long.pivot_table(
    index='시도',  # 행이 될 열
    columns='연도',  # 열이 될 열
    values='인구',  # 값
    aggfunc='sum'  # 중복 시 처리 방법
)

print(df_pivot)

각 형식을 언제 사용하나?

Wide 형식 사용:
  • 연도별, 지역별 비교 표 만들 때.
  • 엑셀로 열어서 보고 싶을 때.
  • 사람이 읽기 쉬운 형태.
Long 형식 사용:
  • groupby(), 필터링 등 분석할 때.
  • 시각화 라이브러리(matplotlib, seaborn)에 전달할 때.
  • pivot_table, groupby 작업 후.
