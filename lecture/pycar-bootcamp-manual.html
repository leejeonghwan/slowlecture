1.1 변수와 자료형.

변수는 데이터를 저장하는 상자입니다. Python에서는 네 가지 기본 자료형이 있습니다.

print(기사_제목)

숫자 (Integer와 Float).

정수(int)는 투표수, 예산액 등입니다. 실수(float)는 소수점이 있는 숫자입니다.

1.6 함수 (Function) 기초.

반복되는 작업을 함수로 만들어 자동화합니다.

Python 
# 금액을 "억 원" 단위로 변환하는 함수
def format_money(원):
    """금액을 읽기 쉽게 포맷팅한다."""
    억 = 원 // 100000000  # 정수 나눗셈
    나머지 = 원 % 100000000

    if 억 > 0 and 나머지 > 0:
        return f"{억}억 {나머지:,}원"
    elif 억 > 0:
        return f"{억}억"
    else:
        return f"{원:,}원"

# 함수 사용
print(format_money(5000000000))  # 50억
print(format_money(3200000000))  # 32억
print(format_money(500000))  # 500,000원

4.1 웹 스크래핑이란?

웹 스크래핑(Web Scraping)은 웹사이트에서 데이터를 자동으로 수집하는 기술입니다. 탐사보도에서는 공공기관의 공개 데이터, 언론사 기사, 정부 공시 정보 등을 효율적으로 수집할 때 사용합니다.

수작업 복붙 vs 자동 수집.

공공데이터포털에서 국회의원 명단 1,000개를 수집한다고 생각해봅시다. 수작업으로 복붙하면 몇 시간이 걸리지만, 웹 스크래핑으로는 몇 초면 됩니다. 또한 수작업은 휴먼 에러(오타, 누락)가 발생하기 쉽습니다.

합법성과 윤리.

웹 스크래핑은 강력한 도구지만, 책임감 있게 사용해야 합니다.

  • robots.txt 확인: 웹사이트의 robots.txt 파일에서 스크래핑을 허용하는지 확인합니다. 예: https://example.com/robots.txt
  • 서버 부하 고려: 짧은 시간에 많은 요청을 보내면 서버가 과부하될 수 있습니다. time.sleep()으로 요청 간격을 띄웁니다.
  • 이용약관 확인: 사이트의 이용약관에서 자동 수집을 금지하는지 확인합니다.
  • 공익 목적: 탐사보도, 학술 연구 등 공익 목적의 데이터 수집은 정당성이 높습니다.
주의: 개인정보, 저작권이 있는 콘텐츠, 금지된 사이트의 데이터는 수집하면 안 됩니다. 법적 문제가 발생할 수 있습니다.

4.2 HTML 구조 이해.

웹 스크래핑을 하려면 먼저 HTML 구조를 이해해야 합니다.

태그, 속성, 중첩 구조.

HTML은 태그로 이루어진 트리 구조입니다. 예를 들어:

HTML 
<table>
  <tr>
    <td class="name">김철수</td>
    <td>민주당</td>
    <td>서울 강남구</td>
  </tr>
</table>

여기서 "class"는 속성이고, "name"은 속성값입니다. 스크래핑할 때 이 속성을 선택자로 사용하여 원하는 요소를 찾습니다.

개발자 도구(F12)로 구조 파악.

페이지네이션 패턴 인식.

많은 사이트에서 URL에 page 파라미터를 포함합니다: https://example.com/news?page=1, https://example.com/news?page=2 등

import time
import requests
from bs4 import BeautifulSoup

all_articles = []

# 1~10 페이지 순회
for page in range(1, 11):
    url = f'https://example.com/news?page={page}'
    response = requests.get(url)
    soup = BeautifulSoup(response.text, 'html.parser')

    # 현재 페이지의 아티클 추출
    articles = soup.find_all('article', class_='news-item')
    for article in articles:
        title = article.find('h2').text.strip()
        link = article.find('a')['href']
        date = article.find('span', class_='date').text.strip()

        all_articles.append({
            'title': title,
            'url': link,
            'date': date
        })

    # 서버 배려: 1초 대기
    time.sleep(1)
    print(f'페이지 {page} 완료')

print(f'총 {len(all_articles)}개 아티클 수집됨')

5.1 API란?

API(Application Programming Interface)는 프로그램 간의 통신 규약입니다. 웹 API를 사용하면 구조화된 데이터를 효율적으로 가져올 수 있습니다.

웹 스크래핑 vs API: 왜 API가 더 나은가.

웹 스크래핑과 API의 차이점을 정리하면:

항목 웹 스크래핑 API
데이터 형식 HTML (구조화되지 않음) JSON, XML (구조화됨)
파싱 난이도 높음 (HTML 구조에 의존) 낮음 (정규화된 형식)
안정성 낮음 (웹사이트 구조 변경 시 깨짐) 높음 (API 호환성 유지)
속도 느림 (전체 HTML 다운로드) 빠름 (필요한 데이터만)
법적 위치 회색지대 명확함 (공식 인터페이스)

REST API 기초 개념.

REST(Representational State Transfer)는 웹 API의 표준입니다. HTTP 메서드를 사용하여 자원에 접근합니다:

  • GET: 데이터 조회 (read-only).
  • POST: 새로운 데이터 생성.
  • PUT: 데이터 수정.
  • DELETE: 데이터 삭제.

탐사보도에서는 주로 GET 요청으로 공개 데이터를 조회합니다.

공공데이터포털 API 소개.

5.2 JSON 구조 이해.

JSON(JavaScript Object Notation)은 API에서 가장 일반적인 데이터 형식입니다. Python의 딕셔너리와 유사합니다.

딕셔너리와 리스트의 조합.

JSON
{ "status": "success", "result": { "total": 2, "items": [ { "name": "김철수", "party": "민주당", "district": "서울 강남구" }, { "name": "이영희", "party": "국민의힘", "district": "부산 해운대구" } ] } }

이 JSON을 Python으로 읽으면:

Python 
import json

json_text = '''
{
  "status": "success",
  "result": {
    "total": 2,
    "items": [...]
  }
}
'''

data = json.loads(json_text)
print(data['status'])  # 'success'
print(data['result']['total'])  # 2
print(data['result']['items'][0]['name'])  # '김철수'

JSON이란?

JSON은 JavaScript Object Notation의 약자로, 웹에서 데이터를 주고받을 때 가장 널리 사용되는 표준 형식입니다. Python의 딕셔너리와 매우 유사한 구조를 가지고 있어 Python 개발자가 쉽게 이해하고 사용할 수 있습니다.

JSON은 키-값 쌍으로 데이터를 표현하며, 배열과 객체를 중첩할 수 있습니다. 예를 들어 한국 뉴스 API의 응답 구조는 다음과 같습니다.

Python의 json 모듈을 사용하면 JSON 데이터를 쉽게 파싱하고 생성할 수 있습니다. 주요 함수는 다음과 같습니다.

JSON과 API 이해하기.

Q1: json.loads()와 json.dumps()의 차이점은 무엇인가요?

답: json.loads()는 JSON 문자열을 Python 객체(딕셔너리)로 변환하고, json.dumps()는 Python 객체를 JSON 문자열로 변환합니다. loads는 "load string", dumps는 "dump string"으로 기억하면 됩니다.

Q2: API 호출 시 params 딕셔너리를 사용하는 이유는 무엇인가요?

답: params를 사용하면 URL에 쿼리 문자열을 자동으로 추가해주므로, 코드가 더 읽기 쉽고 유지보수하기 좋습니다. requests 라이브러리가 자동으로 URL 인코딩도 처리해줍니다.

팁: API 문서 읽는 법.

공공 API를 사용할 때는 반드시 공식 문서를 읽어야 합니다. 문서에서 확인해야 할 사항은 기본 URL(Base URL), 필수 파라미터, 선택 파라미터, 응답 구조, 호출 제한(Rate Limit) 등입니다. 대부분의 공공 API는 공공데이터포털이나 각 기관의 개발자 센터에서 상세한 문서를 제공합니다.

주의: API 호출 제한(Rate Limit).

많은 API는 과도한 호출을 방지하기 위해 시간당 호출 횟수를 제한합니다. 예를 들어 분당 60회 제한이 있다면, 이를 초과할 경우 요청이 거부될 수 있습니다. 대량의 데이터를 수집할 때는 시간을 두고 호출하거나 API 문서에서 권장하는 방식을 따라야 합니다.

API에서 한 번에 제공하는 데이터 개수는 보통 제한되어 있습니다. 전체 데이터를 수집하려면 페이지 단위로 여러 번 요청해야 합니다. 이를 페이지네이션이라고 합니다.

Python 
import requests

url = "https://api.example.com/bills"
all_bills = []
page = 1

# 페이지별로 데이터 수집
while True:
    params = {
        "apiKey": "YOUR_KEY",
        "page": page,
        "pageSize": 100
    }

    response = requests.get(url, params=params)
    data = response.json()

    bills = data["items"]
    if not bills:
        break  # 더 이상 데이터 없음

    all_bills.extend(bills)
    page += 1
    print(f"페이지 {page - 1} 수집 완료")

print(f"총 {len(all_bills)}개의 법안 수집 완료")

퀴즈.

pandas 데이터 분석 이해하기.

Q1: df[df['연봉'] > 10000]과 df.query('연봉 > 10000')의 차이점은 무엇인가요?

답: 둘 다 같은 결과를 반환합니다. 첫 번째는 불린 인덱싱, 두 번째는 query 메서드입니다. query 메서드가 더 읽기 쉽지만, 불린 인덱싱이 더 일반적으로 사용됩니다.

Q2: groupby()와 value_counts()를 언제 각각 사용하나요?

답: value_counts()는 특정 열의 값이 몇 번 나타나는지 세기만 할 때 사용합니다. groupby()는 그룹별로 여러 통계(평균, 합계, 최댓값 등)를 계산할 때 사용합니다. groupby()가 더 강력합니다.

Q3: pd.merge()에서 how='left'와 how='inner'의 차이는?

답: how='inner'는 두 DataFrame에 모두 있는 행만 반환합니다. how='left'는 왼쪽 DataFrame의 모든 행을 유지하고, 오른쪽 DataFrame에 없는 값은 NaN으로 채웁니다.

팁: pandas vs 엑셀 비교표.

pandas는 엑셀과 매우 유사하지만 몇 가지 중요한 차이가 있습니다. 엑셀에서 필터링하는 것은 pandas에서 조건 필터링, 엑셀의 피벗 테이블은 pandas의 groupby와 같습니다. 엑셀은 수동으로 작업하지만, pandas는 코드로 자동화할 수 있다는 게 가장 큰 장점입니다. 대량의 데이터나 반복되는 작업에서 pandas의 강력함이 드러납니다.

pivot_table은 엑셀의 피벗 테이블처럼 데이터를 행과 열로 재배열하고 집계합니다. crosstab은 두 변수의 교차표를 만듭니다.

Python 
# pivot_table: 팀별, 포지션별 평균 연봉
pivot = pd.pivot_table(
    df,
    values='연봉',
    index='팀',
    columns='포지션',
    aggfunc='mean'
)
print(pivot)

# crosstab: 팀과 포지션의 교차표
cross = pd.crosstab(
    df['팀'],
    df['포지션']
)
print(cross)
