과정 개요.

이 실습 교재는 마이크로소프트의 "Generative AI for Beginners" 과정 중 실습 중심 8개 레슨(Lesson 4~11)을 한국 기자와 피디를 위해 특화하여 구성했습니다. 프롬프트 엔지니어링부터 생성형 AI 앱 개발까지 단계별로 학습하며, 즉시 실무에 적용할 수 있는 예제와 코드를 제공합니다.

한국 저널리스트를 위한 학습 목표.

  • AI 도구 활용 능력: ChatGPT, Claude 등 생성형 AI를 기사 작성, 요약, 분석에 활용.
  • 효과적인 프롬프트 작성: 명확한 지시문을 통해 원하는 품질의 결과물 획득.
  • AI 앱 구축 기초: Python과 OpenAI API를 이용한 자동화 도구 개발.
  • 저널리즘 콘텐츠 생성: 기사 요약, 이미지 생성, 팩트체크, 데이터 검색 자동화.
  • AI 윤리 인식: 생성형 AI의 한계와 책임 있는 사용 방법 이해.

프롬프트란 무엇인가?

프롬프트는 생성형 AI에게 주는 지시문입니다. 프롬프트의 품질이 높을수록 더 정확하고 유용한 응답을 얻을 수 있습니다. 좋은 프롬프트를 작성하는 것은 생성형 AI를 효과적으로 활용하는 핵심 기술입니다.

프롬프트의 3가지 구성 요소.

효과적인 프롬프트는 다음 3가지 요소로 이루어집니다:

AI에게 수행해야 할 작업을 명확하게 지시합니다. 모호한 표현보다 구체적이고 직접적인 명령이 좋습니다.

예시:

❌ "뉴스에 대해 말해줘."

✓ "다음 보도자료를 3문장으로 요약하세요."

AI가 처리해야 할 실제 데이터나 텍스트입니다. 기사, 보도자료, 데이터 등 구체적인 입력 자료를 제공합니다.

예시:

보도자료 텍스트: "정부는 오늘 2025년 예산안을 발표했습니다. 총 규모는 600조원으로..."

응답의 형식, 스타일, 길이, 대상 청중 등을 정의합니다. 추가적인 맥락과 제약 조건을 제공합니다.

예시:

- 대상: 일반 독자

- 길이: 3문장

- 스타일: 뉴스 기사 형식

- 용어: 일반 언어 사용, 전문 용어 설명

토큰화(Tokenization) 개념.

LLM은 텍스트를 처리할 때 먼저 텍스트를 "토큰"이라는 작은 단위로 분해합니다. 한국어에서 토큰은 일반적으로 글자 또는 부분 단어입니다.

텍스트
"생성형 AI"
→
토큰화
["생성", "형", " ", "AI"]
→
숫자 변환
[45, 67, 32, 2034]
→
처리
AI 모델 실행

토큰 개수는 비용과 직접 관련됩니다. OpenAI API는 입력 토큰과 출력 토큰 개수에 따라 요금을 청구합니다. 따라서 불필요한 반복을 줄이고 명확하게 지시할수록 비용을 절약할 수 있습니다.

토큰 계산 팁: 일반적으로 한국어 1글자 ≈ 1 토큰, 영어 단어 ≈ 1.3 토큰입니다. OpenAI의 토큰 계산기(platform.openai.com/tokenizer)로 정확히 확인할 수 있습니다.

기본 LLM vs 지시-튜닝 LLM.

LLM에는 두 가지 주요 타입이 있습니다:

기본 LLM (Base LLM)

특징:

  • 패턴 예측만 수행
  • 명령을 따르도록 훈련되지 않음
  • 응답이 예측 불가능할 수 있음

예시 프롬프트: "생성형 AI는"

응답: "생성형 AI는 매우 강력한 기술이며, 많은 산업에서 사용되고 있습니다. 또한..."

지시-튜닝 LLM (Instruction-Tuned)

특징:

  • 사용자 지시를 이해하고 따름
  • 특정 작업(요약, 번역 등)에 최적화
  • 더 안정적이고 예측 가능한 응답

예시 프롬프트: "생성형 AI를 한 문장으로 설명하세요."

응답: "생성형 AI는 대규모 데이터로 훈련된 모델이 텍스트, 이미지, 음성 등을 생성하는 기술입니다."

현재 사용 가능한 대부분의 모델(ChatGPT, GPT-4, Claude 등)은 지시-튜닝된 모델입니다.

프롬프트 실행 (제로샷):

Python
# 국정감사 자료 요약
press_release = """
작년 한 해 동안 우리 부처는 디지털 혁신에 집중했습니다. 특히 공공 데이터 개방,
AI 기술 도입, 클라우드 이전을 추진했으며, 각 분야에서 눈에 띄는 성과를 거두었습니다.
"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "당신은 국회 전문 기자입니다."},
        {"role": "user", "content": f"다음 국정감사 발언을 3문장으로 요약하세요: {press_release}"}
    ],
    temperature=0.3,
    max_tokens=200
)

print(response.choices[0].message.content)

온도(Temperature) 제어.

Temperature는 AI의 응답 다양성을 제어하는 파라미터입니다. 0~1 또는 0~2 범위입니다.

  • 온도 0.0 (~0.3): 예측 가능하고 일관된 응답. 요약, 분류, 팩트 기반 작업에 적합.
  • 온도 0.5 (~0.7): 균형잡힌 응답. 대부분의 일반적인 작업에 적합.
  • 온도 0.8 (~1.0 이상): 창의적이고 다양한 응답. 창작, 브레인스토밍에 적합.

예시 (API에서 온도 설정):

    model="gpt-4o-mini",
    messages=[...],
    temperature=0.2  # 매우 낮음 - 일관성 중심
)

# 창의적인 헤드라인 생성 - 높은 온도
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[...],
    temperature=0.8  # 높음 - 창의성 중심
)

# 균형잡힌 작업 - 중간 온도

    model="gpt-4o-mini",
    messages=[...],
    temperature=0.5  # 중간
)

  • 규칙 기반 (if-then)
  • 예측 가능한 출력
  • 버그 수정 명확

Completion vs ChatCompletion.

OpenAI API는 두 가지 주요 엔드포인트를 제공합니다:

구분 Completion ChatCompletion
입력 텍스트 문자열 메시지 리스트 (role 포함)

토큰 계산 및 모니터링:

Python

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[...]

# 사용한 토큰 정보
print(f"입력 토큰: {response.usage.prompt_tokens}")
print(f"출력 토큰: {response.usage.completion_tokens}")
print(f"총 토큰: {response.usage.total_tokens}")

# 비용 계산 (gpt-4o-mini 기준, 2024년 가격)
# 입력: $0.15 / 1M 토큰
# 출력: $0.60 / 1M 토큰
input_cost = response.usage.prompt_tokens * 0.15 / 1000000
output_cost = response.usage.completion_tokens * 0.60 / 1000000
total_cost = input_cost + output_cost

print(f"예상 비용: ${total_cost:.6f}")

비용 절감 팁:

  • max_tokens 설정: 불필요한 긴 응답 방지.
  • 온도 낮춤: 창의성이 필요 없으면 temperature = 0.3.
  • 배치 처리: 개별 호출보다 배치로 처리 (API 포함된 경우).
  • 캐싱: 반복되는 요청은 결과를 저장.
  • 더 저렴한 모델: gpt-4o-mini 등 경제적 모델 사용.

def summarize_press_release(text, max_words=100):
    """
    보도자료를 요약합니다.

    Args:
        text (str): 보도자료 원문
        max_words (int): 최대 단어 수

    Returns:
        str: 요약된 텍스트

정부 보도자료를 빠르게 읽고 핵심을 추출합니다.

{text}

[요약]"""

        model="gpt-4o-mini",
        messages=[

            {"role": "user", "content": user_prompt}

        temperature=0.3,
        max_tokens=300
    )

    return response.choices[0].message.content

간단한 챗봇 만들기.

메시지 히스토리를 유지하는 기본 챗봇 구현:

Python
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

# 메시지 히스토리 유지
messages = []

def chat(user_input):
    """사용자 입력에 응답하는 챗봇."""

    # 사용자 메시지 추가
    messages.append({"role": "user", "content": user_input})

    # API 호출
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        system="당신은 한국 정치 뉴스를 설명하는 친절한 뉴스봇입니다.",
        messages=messages,
        temperature=0.5
    )

    # 어시스턴트 응답 추가
    assistant_message = response.choices[0].message.content
    messages.append({"role": "assistant", "content": assistant_message})

    return assistant_message

# 간단한 대화 루프
if __name__ == "__main__":
    print("뉴스봇: 안녕하세요! 한국 정치와 정책에 대해 설명해드립니다.")

    while True:
        user_input = input("\n당신: ")
        if user_input.lower() == "exit":
            print("뉴스봇: 감사합니다!")
            break

        response = chat(user_input)
        print(f"뉴스봇: {response}")

Python
# journalist_chatbot.py
# 기자 보조 역할을 하는 대화형 챗봇

from openai import OpenAI
import os
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# 대화 히스토리 초기화
conversation_history = []

# 정치부 기자 보조 시스템 프롬프트
SYSTEM_PROMPT = """당신은 경험 많은 정치부 기자의 보조입니다.
당신의 역할:
- 정치 뉴스의 배경과 맥락을 설명하기
- 주요 정치인과 당의 입장 정리하기
- 기사 작성에 필요한 자료 제공하기
- 관련 법안 및 정책의 상세 설명하기

제약사항:
- 추측하지 말고 확인된 사실만 제공
- 편향된 해석 피하기
- 출처와 맥락 함께 설명

응답 형식:
1. 핵심 요약 (1-2문장)
2. 상세 설명 (불릿 포인트)
3. 추가 조사 제안"""

def chat(user_message):
    """사용자 메시지를 처리하고 AI 응답을 생성합니다."""

    # 사용자 메시지를 히스토리에 추가
    conversation_history.append({
        "role": "user",
        "content": user_message
    })

    # API 호출
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT}
        ] + conversation_history,
        temperature=0.7,
        max_tokens=1000
    )

    # AI 응답 추출
    assistant_message = response.choices[0].message.content

    # 응답을 히스토리에 추가
    conversation_history.append({
        "role": "assistant",
        "content": assistant_message
    })

    return assistant_message

def main():
    """메인 챗봇 루프."""
    print("기자 보조 챗봇에 오신 것을 환영합니다.")
    print("정치 뉴스에 대한 질문을 입력하세요. 'exit'을 입력하면 종료합니다.")
    print("-" * 50)

    while True:
        user_input = input("\n기자: ").strip()

        # 종료 조건
        if user_input.lower() == "exit":
            print("챗봇을 종료합니다.")
            break

        # 공백 입력 무시
        if not user_input:
            continue

        # AI 응답 생성
        print("\nAI: 생각 중...")
        response = chat(user_input)
        print("\nAI: " + response)
        print("-" * 50)

if __name__ == "__main__":
    main()

장시간 대화할 때 토큰 수가 증가하므로, 슬라이딩 윈도우 방식으로 오래된 메시지를 제거합니다.

Python
# sliding_window_context.py
# 토큰 제한에 대응하는 슬라이딩 윈도우 구현

def estimate_tokens(text):
    """텍스트의 토큰 수를 대략적으로 추정합니다."""
    # 일반적으로 1 토큰 ≈ 4자
    return len(text) // 4

def manage_conversation_history(messages, max_tokens=3000):
    """
    대화 히스토리를 관리하여 토큰 제한을 초과하지 않도록 합니다.
    최근 메시지는 유지하고 오래된 메시지부터 제거합니다.
    """

    total_tokens = 0
    managed_messages = []

    # 최근 메시지부터 역순으로 처리
    for message in reversed(messages):
        message_tokens = estimate_tokens(message["content"])

        if total_tokens + message_tokens <= max_tokens:
            managed_messages.insert(0, message)
            total_tokens += message_tokens
        else:
            print(f"⚠️  토큰 제한 도달. {len(messages) - len(managed_messages)}개 오래된 메시지 제거됨.")
            break

    return managed_messages

# 임베딩을 사용한 뉴스 기사 유사도 검색

from openai import OpenAI
import numpy as np

from dotenv import load_dotenv

    "한국은행, 기준금리 3.5%로 동결 결정",
    "미 연준 금리 인하 시사에 원달러 환율 급락",
    "삼성전자 3분기 영업이익 전년比 30% 증가",
    "정부, 내년도 예산안 656조원 편성",
    "서울 아파트 매매가 3개월 연속 상승"
]

def get_embedding(text):
    """텍스트를 임베딩 벡터로 변환합니다."""
    response = client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    )
    return np.array(response.data[0].embedding)

def cosine_similarity(vec1, vec2):
    """두 벡터 간의 코사인 유사도를 계산합니다."""
    return np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))

def rag_query(question):
    """
    RAG 파이프라인:
    1. 질문과 관련된 기사 검색
    2. 검색된 기사를 컨텍스트로 사용하여 답변 생성
    """

    print(f"[2단계] 질문 '{question}'과 관련된 기사 검색...")

    # Retrieval: 관련 기사 검색
    search_results = collection.query(
        query_texts=[question],
        n_results=2
    )

    # 검색 결과 표시
    retrieved_docs = []
    for doc, metadata in zip(
        search_results["documents"][0],
        search_results["metadatas"][0]
    ):
        retrieved_docs.append({
            "title": metadata["title"],
            "content": doc
        })
        print(f"  - {metadata['title']}")

    print("\n[3단계] 검색된 기사를 기반으로 답변 생성 중...")

    # Generation: 컨텍스트를 포함한 답변 생성
    context = "\n".join([
        f"기사: {doc['title']}\n본문: {doc['content']}"
        for doc in retrieved_docs
    ])

    response = openai_client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {
                "role": "system",
                "content": "당신은 뉴스 기사를 기반으로 정확한 답변을 제공하는 기자 보조입니다. 주어진 기사의 정보만 사용하여 답변하세요."
            },
            {
                "role": "user",
                "content": f"다음 기사들을 바탕으로 질문에 답해주세요.\n\n[기사 정보]\n{context}\n\n[질문]\n{question}"
            }
        ],
        temperature=0.5,
        max_tokens=500
    )

    return response.choices[0].message.content

def generate_news_illustration(article_title, article_content=None):
    """
    뉴스 기사 제목을 기반으로 삽화를 생성합니다.

    Args:
        article_title: 뉴스 기사 제목

    else:

이 뉴스에 어울리는 전문적인 삽화를 생성해주세요.
스타일: 현대적인 신문 일러스트레이션

from openai import OpenAI
import requests
from PIL import Image, ImageDraw
import io
import os
from dotenv import load_dotenv

load_dotenv()

def create_circular_mask(image_size=1024, mask_radius=200):
    """
    원형 마스크를 생성합니다.
    마스크 영역(검은색)이 편집할 부분입니다.
    """
    mask = Image.new("RGBA", (image_size, image_size), (255, 255, 255, 255))
    draw = ImageDraw.Draw(mask)

    # 중앙에 검은 원 그리기 (편집 영역)
    center = image_size // 2
    draw.ellipse(
        [center - mask_radius, center - mask_radius,
         center + mask_radius, center + mask_radius],
        fill=(0, 0, 0, 0)
    )

    return mask

def create_rectangular_mask(image_size=1024, box_coords=(256, 256, 768, 768)):
    """
    직사각형 마스크를 생성합니다.
    """
    mask = Image.new("RGBA", (image_size, image_size), (255, 255, 255, 255))
    draw = ImageDraw.Draw(mask)

    # 직사각형 영역을 투명하게 (편집 영역)
    draw.rectangle(box_coords, fill=(0, 0, 0, 0))

    return mask

def download_image_as_pil(url):
    """URL의 이미지를 PIL Image로 다운로드합니다."""
    response = requests.get(url)
    image = Image.open(io.BytesIO(response.content))
    return image

def edit_image(image_path, edit_prompt, mask_type="circular"):
    """
    기존 이미지의 일부를 편집합니다.

    Args:
        image_path: 편집할 이미지 경로
        edit_prompt: 편집 내용 설명
        mask_type: 마스크 타입 ('circular' 또는 'rectangular')
    """

    if not os.path.exists(image_path):
        print(f"⚠️  이미지 파일을 찾을 수 없습니다: {image_path}")
        return None

    # 이미지와 마스크 준비
    image = Image.open(image_path).resize((1024, 1024))

    if mask_type == "circular":
        mask = create_circular_mask()
    else:
        mask = create_rectangular_mask()

    # 임시 파일로 저장
    temp_image = "/tmp/temp_image.png"
    temp_mask = "/tmp/temp_mask.png"
    image.save(temp_image)
    mask.save(temp_mask)

    print(f"[편집] '{edit_prompt}' 적용 중...")

    # DALL-E 편집 API 호출
    response = client.images.edit(
        image=open(temp_image, "rb"),
        mask=open(temp_mask, "rb"),
        prompt=edit_prompt,
        n=1,
        size="1024x1024"
    )

    # 임시 파일 정리
    os.remove(temp_image)
    os.remove(temp_mask)

    print("✓ 편집 완료")
    return response.data[0].url

def create_image_variation(image_path):
    """
    기존 이미지를 기반으로 변형된 이미지를 생성합니다.
    """

    if not os.path.exists(image_path):
        print(f"⚠️  이미지 파일을 찾을 수 없습니다: {image_path}")
        return None

    print("[변형] 이미지 변형 생성 중...")

    # 이미지 크기 조정 (1024x1024만 지원)
    image = Image.open(image_path).resize((1024, 1024))

        image=open("/tmp/temp_variation.png", "rb"),
        n=1,
        size="1024x1024"
    )

    print("✓ 변형 생성 완료")
    return response.data[0].url

import gradio as gr
from openai import OpenAI
import os
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def summarize_article(article_text, style):
    """보도자료를 선택한 스타일로 자동 요약합니다."""
    styles = {
        "속보 (30자)": "뉴스 속보로 30자 이내 한 줄로 정리하세요. 제목 없음, 본문만.",
        "기사 (300자)": "신문 기사 형식으로 300자 이내. 구성: [제목]\n[리드]\n[본문]. 헤드라인은 15자 이내.",
        "SNS (140자)": "SNS용 140자 이내 요약. 해시태그 2-3개 포함. 이모지 선택적.",
        "브리핑 (5줄)": "불릿 포인트 5개. 각 줄 20자 이내. 핵심만 추출."
    }

    if not article_text.strip():
        return "오류: 보도자료를 입력하세요."

    prompt_template = styles.get(style, styles["기사 (300자)"])

    try:
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {
                    "role": "system",
                    "content":
                        "당신은 한국 뉴스 편집 데스크 에디터입니다. "
                        "정확한 정보 전달과 간결함을 최우선으로 합니다. "
                        "요청된 형식을 정확히 지켜주세요."
                },
                {
                    "role": "user",
                    "content": f"{prompt_template}\n\n보도자료:\n{article_text}"
                }
            ],
            temperature=0.3,
            max_tokens=500
        )
        return response.choices[0].message.content
    except Exception as e:
        return f"API 오류: {str(e)}"

def process_all_styles(article_text):
    """모든 스타일로 동시에 요약합니다."""
    if not article_text.strip():
        return "오류", "오류", "오류", "오류"

    summary_map = {
        "속보 (30자)": summarize_article(article_text, "속보 (30자)"),
        "기사 (300자)": summarize_article(article_text, "기사 (300자)"),
        "SNS (140자)": summarize_article(article_text, "SNS (140자)"),
        "브리핑 (5줄)": summarize_article(article_text, "브리핑 (5줄)")
    }
    return (
        summary_map["속보 (30자)"],
        summary_map["기사 (300자)"],
        summary_map["SNS (140자)"],
        summary_map["브리핑 (5줄)"]

import gradio as gr
from openai import OpenAI
import json
import os
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def analyze_sentiment_batch(comments_text):
    """여러 댓글을 한 번에 감성 분석합니다."""
    if not comments_text.strip():
        return "오류: 댓글을 입력하세요", {}, ""

    # 댓글을 라인별로 분리
    comments = [c.strip() for c in comments_text.split("\n") if c.strip()]

    try:
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {
                    "role": "system",
                    "content":
                        "당신은 한국 SNS 감성 분석 전문가입니다. "
                        "각 댓글을 긍정(positive), 부정(negative), 중립(neutral) 중 하나로 분류하고, "
                        "주요 감정 키워드 2-3개를 추출하세요. "
                        "JSON 형식으로 응답하세요: "
                        "[{\"comment\": \"...\", \"sentiment\": \"...\", \"keywords\": [\"...\", \"...\"]}]"
                },
                {
                    "role": "user",
                    "content": f"다음 댓글들을 분석하세요:\n{json.dumps(comments, ensure_ascii=False)}"
                }
            ],
            temperature=0.5,
            max_tokens=1000
        )

        result_text = response.choices[0].message.content
        # JSON 파싱
        results = json.loads(result_text)

        # 통계 계산
        sentiment_count = {"positive": 0, "negative": 0, "neutral": 0}
        all_keywords = []

        for r in results:
            sentiment_count[r["sentiment"]] += 1
            all_keywords.extend(r.get("keywords", []))

        # 결과 텍스트 조합
        summary = f"""
📊 감성 분석 결과:
- 긍정 (😊): {sentiment_count['positive']}개
- 부정 (😞): {sentiment_count['negative']}개
- 중립 (😐): {sentiment_count['neutral']}개

🔑 주요 키워드: {', '.join(set(all_keywords))[:200]}
        """

        # 그래프 데이터
        chart_data = {
            "긍정": sentiment_count["positive"],
            "부정": sentiment_count["negative"],
            "중립": sentiment_count["neutral"]
        }

        # 상세 결과
        details = ""
        for r in results:
            emoji = {
                "positive": "😊",
                "negative": "😞",
                "neutral": "😐"
            }[r["sentiment"]]
            details += f"{emoji} {r['comment'][:60]}... ({r['sentiment']})\n"

        return summary, chart_data, details

    except Exception as e:
        return f"분석 오류: {str(e)}", {}, ""

Function Calling 연동.

학습 목표: OpenAI의 Function Calling과 JSON 구조화 모드를 사용해 LLM이 자동으로 외부 함수/API를 호출하고, 실시간 데이터(주가, 날씨, 검색 결과)를 기반으로 답변하는 뉴스룸 시스템을 구축한다.

Lesson 11은 AI 모델이 도구를 자동으로 선택 및 실행하는 능력을 다룹니다. Function Calling을 통해 LLM은 사용자 질문을 분석해 "아, 이건 날씨 함수가 필요하네" 또는 "이건 데이터베이스 검색이 필요하네"라고 판단하고 자동 실행합니다. 뉴스룸에서 기자가 "삼성전자 주가는?"이라고 물으면, AI가 자동으로 stock_price 함수를 호출하는 식입니다.

# ============ Step 1: 함수 정의 (JSON 스키마) ============

def get_weather(city, unit="celsius"):
    """실제 날씨 데이터 조회 함수 (백엔드)."""
    data = weather_db.get(city)

    if not data:

            "error": f"'{city}' 날씨 정보를 찾을 수 없습니다",
            "supported_cities": list(weather_db.keys())
        }, ensure_ascii=False)

    # 화씨 변환
    temp = data["temp"]
    if unit == "fahrenheit":
        temp = temp * 9 / 5 + 32

        "city": city,
        "temperature": temp,
        "unit": unit,
        "condition": data["condition"],
        "humidity": data["humidity"],
        "wind": data["wind"]
    }, ensure_ascii=False)

def chat_with_functions(user_message):

    # 첫 번째 요청: LLM에게 질문
    messages = [{"role": "user", "content": user_message}]

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        tools=tools,
        tool_choice="auto"  # LLM이 자동으로 함수 선택
    )

    # 함수 호출 여부 확인
    if response.choices[0].finish_reason == "tool_calls":
        print(f"⚙️ LLM이 함수 호출을 요청했습니다.")

        tool_call = response.choices[0].message.tool_calls[0]
        func_name = tool_call.function.name
        func_args = json.loads(tool_call.function.arguments)

        print(f"📞 함수 호출: {func_name}({func_args})")

        # 함수 실행
        if func_name == "get_weather":
            result = get_weather(**func_args)

        print(f"✅ 함수 반환값: {result[:100]}...")

        # 두 번째 요청: 함수 결과를 LLM에 전달
        messages.append(response.choices[0].message)
        messages.append({
            "role": "tool",
            "tool_call_id": tool_call.id,
            "content": result
        })

        # 최종 답변 생성
        final_response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=messages
        )

        final_answer = final_response.choices[0].message.content
        print(f"\n🎯 최종 답변:\n{final_answer}")

        return final_answer

    else:
        # 함수 호출 없이 직답
        answer = response.choices[0].message.content
        print(f"\n🎯 직접 답변:\n{answer}")
        return answer
