과정 개요.

이 실습 교재는 마이크로소프트의 "Generative AI for Beginners" 과정 중 실습 중심 8개 레슨(Lesson 4~11)을 한국 기자와 피디를 위해 특화하여 구성했습니다. 프롬프트 엔지니어링부터 생성형 AI 앱 개발까지 단계별로 학습하며, 즉시 실무에 적용할 수 있는 예제와 코드를 제공합니다.

한국 저널리스트를 위한 학습 목표.

  • AI 도구 활용 능력: ChatGPT, Claude 등 생성형 AI를 기사 작성, 요약, 분석에 활용.
  • 효과적인 프롬프트 작성: 명확한 지시문을 통해 원하는 품질의 결과물 획득.
  • AI 앱 구축 기초: Python과 OpenAI API를 이용한 자동화 도구 개발.
  • 저널리즘 콘텐츠 생성: 기사 요약, 이미지 생성, 팩트체크, 데이터 검색 자동화.
  • AI 윤리 인식: 생성형 AI의 한계와 책임 있는 사용 방법 이해.

프롬프트란 무엇인가?

프롬프트는 생성형 AI에게 주는 지시문입니다. 프롬프트의 품질이 높을수록 더 정확하고 유용한 응답을 얻을 수 있습니다. 좋은 프롬프트를 작성하는 것은 생성형 AI를 효과적으로 활용하는 핵심 기술입니다.

프롬프트의 3가지 구성 요소.

효과적인 프롬프트는 다음 3가지 요소로 이루어집니다:

AI에게 수행해야 할 작업을 명확하게 지시합니다. 모호한 표현보다 구체적이고 직접적인 명령이 좋습니다.

예시:

❌ "뉴스에 대해 말해줘."

✓ "다음 보도자료를 3문장으로 요약하세요."

AI가 처리해야 할 실제 데이터나 텍스트입니다. 기사, 보도자료, 데이터 등 구체적인 입력 자료를 제공합니다.

예시:

보도자료 텍스트: "정부는 오늘 2025년 예산안을 발표했습니다. 총 규모는 600조원으로..."

응답의 형식, 스타일, 길이, 대상 청중 등을 정의합니다. 추가적인 맥락과 제약 조건을 제공합니다.

예시:

- 대상: 일반 독자

- 길이: 3문장

- 스타일: 뉴스 기사 형식

- 용어: 일반 언어 사용, 전문 용어 설명

토큰화(Tokenization) 개념.

LLM은 텍스트를 처리할 때 먼저 텍스트를 "토큰"이라는 작은 단위로 분해합니다. 한국어에서 토큰은 일반적으로 글자 또는 부분 단어입니다.

텍스트
"생성형 AI"
→
토큰화
["생성", "형", " ", "AI"]
→
숫자 변환
[45, 67, 32, 2034]
→
처리
AI 모델 실행

토큰 개수는 비용과 직접 관련됩니다. OpenAI API는 입력 토큰과 출력 토큰 개수에 따라 요금을 청구합니다. 따라서 불필요한 반복을 줄이고 명확하게 지시할수록 비용을 절약할 수 있습니다.

토큰 계산 팁: 일반적으로 한국어 1글자 ≈ 1 토큰, 영어 단어 ≈ 1.3 토큰입니다. OpenAI의 토큰 계산기(platform.openai.com/tokenizer)로 정확히 확인할 수 있습니다.

기본 LLM vs 지시-튜닝 LLM.

LLM에는 두 가지 주요 타입이 있습니다:

기본 LLM (Base LLM)

특징:

  • 패턴 예측만 수행
  • 명령을 따르도록 훈련되지 않음
  • 응답이 예측 불가능할 수 있음

예시 프롬프트: "생성형 AI는"

응답: "생성형 AI는 매우 강력한 기술이며, 많은 산업에서 사용되고 있습니다. 또한..."

지시-튜닝 LLM (Instruction-Tuned)

특징:

  • 사용자 지시를 이해하고 따름
  • 특정 작업(요약, 번역 등)에 최적화
  • 더 안정적이고 예측 가능한 응답

예시 프롬프트: "생성형 AI를 한 문장으로 설명하세요."

응답: "생성형 AI는 대규모 데이터로 훈련된 모델이 텍스트, 이미지, 음성 등을 생성하는 기술입니다."

현재 사용 가능한 대부분의 모델(ChatGPT, GPT-4, Claude 등)은 지시-튜닝된 모델입니다.

프롬프트 실행 (제로샷):

Python
# 국정감사 자료 요약
press_release = """
작년 한 해 동안 우리 부처는 디지털 혁신에 집중했습니다. 특히 공공 데이터 개방,
AI 기술 도입, 클라우드 이전을 추진했으며, 각 분야에서 눈에 띄는 성과를 거두었습니다.
"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "당신은 국회 전문 기자입니다."},
        {"role": "user", "content": f"다음 국정감사 발언을 3문장으로 요약하세요: {press_release}"}
    ],
    temperature=0.3,
    max_tokens=200
)

print(response.choices[0].message.content)

온도(Temperature) 제어.

Temperature는 AI의 응답 다양성을 제어하는 파라미터입니다. 0~1 또는 0~2 범위입니다.

  • 온도 0.0 (~0.3): 예측 가능하고 일관된 응답. 요약, 분류, 팩트 기반 작업에 적합.
  • 온도 0.5 (~0.7): 균형잡힌 응답. 대부분의 일반적인 작업에 적합.
  • 온도 0.8 (~1.0 이상): 창의적이고 다양한 응답. 창작, 브레인스토밍에 적합.

예시 (API에서 온도 설정):

    model="gpt-4o-mini",
    messages=[...],
    temperature=0.2  # 매우 낮음 - 일관성 중심
)

# 창의적인 헤드라인 생성 - 높은 온도
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[...],
    temperature=0.8  # 높음 - 창의성 중심
)

# 균형잡힌 작업 - 중간 온도

    model="gpt-4o-mini",
    messages=[...],
    temperature=0.5  # 중간
)

토큰 관리와 비용 최적화.

API 사용 시 토큰 개수에 따라 비용이 발생합니다. 효율적 관리가 필수입니다.

토큰 계산 및 모니터링:

Python
# API 응답에서 토큰 정보 추출
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[...]
)

# 사용한 토큰 정보
print(f"입력 토큰: {response.usage.prompt_tokens}")
print(f"출력 토큰: {response.usage.completion_tokens}")
print(f"총 토큰: {response.usage.total_tokens}")

# 비용 계산 (gpt-4o-mini 기준, 2024년 가격)
# 입력: $0.15 / 1M 토큰
# 출력: $0.60 / 1M 토큰
input_cost = response.usage.prompt_tokens * 0.15 / 1000000
output_cost = response.usage.completion_tokens * 0.60 / 1000000
total_cost = input_cost + output_cost

print(f"예상 비용: ${total_cost:.6f}")

비용 절감 팁:

  • max_tokens 설정: 불필요한 긴 응답 방지.
  • 온도 낮춤: 창의성이 필요 없으면 temperature = 0.3.
  • 배치 처리: 개별 호출보다 배치로 처리 (API 포함된 경우).
  • 캐싱: 반복되는 요청은 결과를 저장.
  • 더 저렴한 모델: gpt-4o-mini 등 경제적 모델 사용.

from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

# 보도자료 요약 함수
def summarize_press_release(text, max_words=100):
    """
    보도자료를 요약합니다.

    Args:
        text (str): 보도자료 원문
        max_words (int): 최대 단어 수

    Returns:
        str: 요약된 텍스트
    """

    system_prompt = """당신은 경험 많은 뉴스 기자입니다.
정부 보도자료를 빠르게 읽고 핵심을 추출합니다.
쉽고 명확한 한국어를 사용합니다."""

    user_prompt = f"""다음 보도자료를 {max_words}단어 이내로 요약하세요.
핵심 사건, 수치, 인물을 포함하세요.

[보도자료]
{text}

[요약]"""

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt}
        ],
        temperature=0.3,
        max_tokens=300
    )

    return response.choices[0].message.content

if __name__ == "__main__":

간단한 챗봇 만들기.

메시지 히스토리를 유지하는 기본 챗봇 구현:

Python
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

# 메시지 히스토리 유지
messages = []

def chat(user_input):
    """사용자 입력에 응답하는 챗봇."""

    # 사용자 메시지 추가
    messages.append({"role": "user", "content": user_input})

    # API 호출
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        system="당신은 한국 정치 뉴스를 설명하는 친절한 뉴스봇입니다.",
        messages=messages,
        temperature=0.5
    )

    # 어시스턴트 응답 추가
    assistant_message = response.choices[0].message.content
    messages.append({"role": "assistant", "content": assistant_message})

    return assistant_message

# 간단한 대화 루프
if __name__ == "__main__":
    print("뉴스봇: 안녕하세요! 한국 정치와 정책에 대해 설명해드립니다.")

    while True:
        user_input = input("\n당신: ")
        if user_input.lower() == "exit":
            print("뉴스봇: 감사합니다!")
            break

        response = chat(user_input)
        print(f"뉴스봇: {response}")

SDK/API 활용 개발 전략.

OpenAI, Claude, Google Gemini 등 LLM API를 활용하여 신뢰성 있는 채팅 앱을 빠르게 구축할 수 있습니다. 각 API는 고유한 특성이 있으므로, 프로젝트 요구사항에 맞는 모델을 선택해야 합니다.

API 방식의 장점: 비용 효율적, 빠른 개발, 모델 업데이트 자동 적용, 인프라 관리 불필요.

자체 구축의 장점: 완전한 제어, 데이터 프라이버시, 커스터마이징 자유도.

한국 뉴스룸의 경우 API 방식으로 시작하여 트래픽 증가 시 자체 모델 도입 검토가 권장됩니다.

대화 컨텍스트와 메모리 관리.

AI 채팅 앱의 가장 중요한 특성은 이전 대화를 기억하는 것입니다. 시스템 메시지, 사용자 메시지, 어시스턴트 응답을 순차적으로 저장하여 일관된 대화를 유지합니다.

메모리 관리 전략: 토큰 수가 증가하면 비용이 올라가므로, 중요한 대화만 보존하거나 요약하여 토큰 효율을 높일 수 있습니다.

시스템 메시지 프레임워크.

시스템 메시지는 AI의 동작 방식을 결정하는 핵심입니다. 다음 요소를 포함해야 합니다.

  • 역할(Role): "당신은 한국 뉴스룸의 취재 도우미입니다."
  • 제약(Constraints): "정치적 편견을 피하고 객관적이어야 합니다."
  • 출력 형식(Output Format): "답변은 마크다운 형식으로 제공하세요."
  • 기술(Techniques): "답이 불명확한 경우 명확화를 요청하세요."

UX 고려사항.

사용자 경험을 향상시키기 위해 다음을 고려합니다.

  • 모호한 답변 처리: AI가 확신하지 못하면 "제가 정확히 이해하지 못했습니다. 다시 설명해주시겠어요?"라고 요청.
  • 명확화 요청: 사용자의 의도를 다시 확인하는 버튼 제공.
  • 응답 시간: 3초 이상 지연 시 로딩 표시기 표시.
  • 오류 처리: API 오류 시 사용자 친화적 메시지 표시.

접근성 지원 (Accessibility).

모든 사용자가 채팅 앱을 사용할 수 있도록 장애 지원을 포함해야 합니다.

  • 시각 장애: 스크린 리더 호환, alt 속성, 명확한 색 대비.
  • 청각 장애: 음성 응답에 텍스트 자막 제공.
  • 운동 장애: 키보드 네비게이션, 큰 클릭 영역.
  • 인지 장애: 간단한 언어, 일관된 레이아웃, 단계별 설명.

품질 지표 (Quality Metrics).

채팅 앱의 성능을 측정하기 위해 다음 지표를 모니터링합니다.

  • 응답 시간 (Response Time): 사용자 입력 후 첫 응답까지의 시간 (목표: 2초 이하).
  • 정확도 (Accuracy): 정답의 비율 (목표: 85% 이상).
  • F1 스코어: 정밀도와 재현율의 조화평균 (0-1, 높을수록 좋음).
  • 사용자 만족도 (User Satisfaction): 설문 조사 또는 좋아요/싫어요 버튼 (목표: 4.0/5.0 이상).

Example: 뉴스룸 취재 도우미 챗봇.

실제 한국 뉴스룸에서 사용할 수 있는 취재 보조 챗봇을 구축해봅시다.

Python
from openai import OpenAI

client = OpenAI()

# 대화 메모리 초기화. 시스템 메시지로 역할 정의.
conversation = [
    {
        "role": "system",
        "content": "당신은 한국 뉴스룸의 취재 도우미입니다. 기자들의 취재 질문에 답하고, 관련 배경 정보를 제공합니다. 정치적 편견을 피하고 객관적이어야 합니다."
    },
]

# 대화 루프 시작.
while True:
    user_input = input("기자: ")

    # 종료 조건.
    if user_input.lower() == "종료":
        break

    # 사용자 메시지를 대화 히스토리에 추가.
    conversation.append({
        "role": "user",
        "content": user_input
    })

    # API 호출로 AI 응답 생성.
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=conversation,
        temperature=0.7
    )

    # 응답 추출.
    reply = response.choices[0].message.content

    # 대화 히스토리에 어시스턴트 응답 추가.
    conversation.append({
        "role": "assistant",
        "content": reply
    })

    # 답변 출력.
    print(f"AI: {reply}")

텍스트 임베딩 (Text Embedding).

임베딩은 텍스트를 숫자 벡터로 변환하는 과정입니다. 의미가 비슷한 텍스트는 벡터 공간에서 가까이 위치합니다.

임베딩의 의미: 텍스트 "서울 시장 재선거"와 "서울시 행정 개편"은 서로 다른 주제지만, 모두 "서울"이라는 벡터 성분을 공유합니다. 이렇게 의미를 숫자로 표현하면 컴퓨터가 의미를 이해할 수 있습니다.

코사인 유사도 (Cosine Similarity).

두 벡터 간의 유사도를 측정하는 방법입니다. 0과 1 사이의 값을 가지며, 1에 가까울수록 더 유사합니다.

공식: similarity = (A · B) / (||A|| × ||B||)

예시: 벡터 A = [0.5, 0.2], B = [0.48, 0.22]

similarity = (0.5×0.48 + 0.2×0.22) / (√(0.5²+0.2²) × √(0.48²+0.22²)) ≈ 0.997

매우 높은 유사도(0.997)는 두 텍스트가 매우 관련이 있음을 의미합니다.

임베딩 생성 과정.

OpenAI의 text-embedding-ada-002 모델을 사용하여 임베딩을 생성합니다. 이 모델은 1,536차원의 벡터를 생성하며, 고성능과 저비용의 최적 균형을 제공합니다.

임베딩 비용: ada-002는 매우 저렴합니다. 100만 토큰당 약 $0.02로 책정되어 있어, 대규모 문서 아카이브도 경제적으로 처리 가능합니다.

Example: 기사 아카이브 시맨틱 검색.

한국 뉴스 아카이브에서 주제별로 관련 기사를 찾는 시스템을 구축합니다.

Python
from openai import OpenAI
import numpy as np

client = OpenAI()

# 뉴스룸의 기사 샘플.
articles = [
    "국회 본회의에서 예산안이 통과되었다. 정부 부처별 예산 현황을 분석했습니다.",
    "헌법재판소가 탄핵 심판 결정을 발표했다. 정치적 파장이 예상됩니다.",
    "한국은행이 기준금리를 동결했다. 금리 인상 전망이 연기되었습니다."
]

# 임베딩 생성 함수.
def get_embedding(text):
    response = client.embeddings.create(
        model="text-embedding-ada-002",
        input=text
    )
    return response.data[0].embedding

# 기사 임베딩 생성 및 저장.
article_embeddings = []
for article in articles:
    embedding = get_embedding(article)
    article_embeddings.append({
        "text": article,
        "embedding": embedding
    })

# 검색 쿼리 임베딩.
query = "금리 정책 관련 뉴스"
query_embedding = get_embedding(query)

# 코사인 유사도 함수.
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 유사도 계산 및 정렬.
results = []
for item in article_embeddings:
    score = cosine_similarity(query_embedding, item["embedding"])
    results.append({
        "text": item["text"],
        "score": score
    })

# 유사도 높은 순서대로 정렬.
results.sort(key=lambda x: x["score"], reverse=True)

# 상위 결과 출력.
for i, result in enumerate(results[:2]):
    print(f"{i+1}. 유사도: {result['score']:.3f}")
    print(f"   {result['text']}")

실습 워크플로우.

벡터 검색 시스템을 구축하는 5단계 프로세스입니다.

  1. 문서 수집 (Collection): 뉴스 아카이브에서 기사 텍스트 추출.
  2. 청킹 (Chunking): 긴 기사를 500자 정도의 의미 있는 단위로 분할.
  3. 임베딩 (Embedding): 각 청크를 벡터로 변환.
  4. 저장 (Storage): 벡터와 원본 텍스트를 벡터 DB에 저장.
  5. 검색 (Search): 사용자 쿼리를 임베딩하고 코사인 유사도로 매칭.

모듈 6: 이미지 생성 앱 만들기.

DALL-E를 활용한 AI 이미지 생성 기술과 안전한 활용 방법을 학습합니다.

DALL-E 기술 기반.

DALL-E는 두 가지 주요 기술을 결합합니다.

CLIP (Contrastive Language-Image Pre-training): 텍스트와 이미지 간의 의미 관계를 학습합니다. 프롬프트를 이해하는 핵심 역할을 합니다.

Diffusion 모델: 무작위 노이즈에서 시작하여 점진적으로 이미지를 생성합니다. 여러 단계를 거쳐 정교한 디테일이 추가됩니다.

결과적으로 자연스럽고 의미 있는 이미지가 생성되며, 기존 생성형 모델보다 품질이 우수합니다.

이미지 생성 파라미터.

DALL-E API 호출 시 제어할 수 있는 주요 파라미터입니다.

  • prompt (필수): 이미지 설명. 상세할수록 좋은 결과를 얻습니다. (최대 4,000자)
  • size: 이미지 크기. 옵션: 256x256, 512x512, 1024x1024 (기본값: 1024x1024).
  • n: 생성할 이미지 개수. 1~10 (기본값: 1).
  • quality: 이미지 품질. "standard" 또는 "hd" (기본값: standard).
  • style: 스타일 지정. "natural" 또는 "vivid" (기본값: vivid).

메타 프롬프트 (Meta-Prompt).

안전하고 적절한 이미지 생성을 위해 시스템 프롬프트를 설정합니다.

중요: 메타 프롬프트는 부적절한 콘텐츠 생성을 방지합니다. 폭력, 혐오, 저작권 침해 이미지 생성을 명시적으로 금지해야 합니다.

권장 메타 프롬프트:

"뉴스 기사를 위한 일러스트레이션을 생성하세요. 다음을 금지합니다: (1) 실제 인물의 얼굴, (2) 폭력적 콘텐츠, (3) 저작권 보호 캐릭터. 모든 이미지는 뉴스 매체 기준의 윤리 검수를 통과해야 합니다."

이미지 편집과 변형.

생성된 이미지를 후처리하는 기능들입니다.

  • 이미지 편집 (Image Editing): 마스크 영역에 대해 다시 생성 (inpainting).
  • 이미지 변형 (Image Variation): 기존 이미지와 유사하지만 다른 스타일의 이미지 생성.
  • 이미지 크기 조정 (Image Resizing): 생성된 이미지를 다양한 형식으로 변환.

Temperature 제어.

DALL-E의 temperature 파라미터는 생성 결과의 다양성을 제어합니다.

  • Temperature = 0: 완전히 결정적. 같은 프롬프트는 항상 동일한 이미지 생성.
  • Temperature = 0.5: 균형잡힌 설정. 일관성과 다양성의 중간.
  • Temperature = 1.0: 매우 창의적. 매번 완전히 다른 이미지 생성.
권장사항: 뉴스 기사의 일관된 일러스트레이션을 원하면 0.3~0.5를 사용합니다.

Example: 뉴스 기사용 일러스트레이션 자동 생성.

한국 뉴스 기사의 제목으로 관련 일러스트를 자동 생성하는 시스템입니다.

Python
from openai import OpenAI
from PIL import Image
import requests
import os
from io import BytesIO

client = OpenAI()

# 뉴스 제목 샘플.
article_title = "서울 국회의사당 앞에서 시민들 평화 집회"

# 제목으로부터 프롬프트 생성.
prompt = f"서울 국회의사당 앞에서 시민들이 평화롭게 집회하는 모습, 수채화 스타일, 하늘색 톤"

# 메타 프롬프트: 부적절한 콘텐츠 방지.
meta_prompt = "뉴스 기사용 일러스트입니다. 실제 인물의 얼굴, 폭력성, 저작권 침해를 피하세요."

# 이미지 생성 API 호출.
response = client.images.generate(
    model="dall-e-3",
    prompt=prompt,
    size="1024x1024",
    n=1,
    quality="standard"
)

# 생성된 이미지 URL 추출.
image_url = response.data[0].url

# 이미지 다운로드.
img_response = requests.get(image_url)
img = Image.open(BytesIO(img_response.content))

# 파일 저장.
output_path = "news_illustration.png"
img.save(output_path)

print(f"이미지 저장됨: {output_path}")
print(f"원본 URL: {image_url}")

로우코드 개발이란 무엇인가?

로우코드(Low-code) 개발은 최소한의 코딩으로 비즈니스 애플리케이션을 빠르게 구축하는 방식입니다. 드래그앤드롭 인터페이스와 사전 빌드된 컴포넌트를 사용하여 전문 개발자가 아닌 사람도 앱을 만들 수 있습니다. 뉴스룸에서는 기사 관리, 보도자료 분류, 데이터 분석을 자동화할 수 있습니다.

핵심: 로우코드는 빠른 프로토타입 개발, 비용 절감, 비기술 팀원도 참여 가능한 환경을 제공합니다.

Microsoft Power Platform 구성.

Microsoft는 네 가지 주요 도구로 Power Platform을 구성했습니다.

Power Apps: 캔버스 앱(자유로운 UI 디자인) 또는 모델 기반 앱(데이터 중심)을 만듭니다.

Power Automate: 워크플로우 자동화 도구입니다. "A 이벤트 발생 → B 자동 실행" 형태의 연결을 만듭니다.

Power BI: 데이터 시각화 및 대시보드 도구입니다.

Copilot Studio: 자연어로 챗봇과 AI 에이전트를 만듭니다.

Copilot으로 앱 자동 생성하기.

Microsoft의 가장 혁신적인 기능은 Copilot입니다. 자연어로 앱이 필요한 목적을 설명하면 Copilot이 기본 앱 구조를 자동으로 생성합니다.

모듈 8: Function Calling 연동.

LLM이 외부 데이터와 API에 접근하는 핵심 메커니즘.

Function Calling이 해결하는 문제.

LLM을 실무에 사용할 때 두 가지 근본적인 문제가 있습니다.

문제 1: 응답 포맷의 일관성.

사용자: "지난주 경제 뉴스 찾아줘." → LLM: "지난주에는 여러 경제 뉴스가 있었습니다. 주식이... (자유로운 텍스트)" 매번 다른 형태의 응답이 나와서 프로그램이 처리하기 어렵습니다.

문제 2: 현재 데이터에 접근 불가.

LLM은 학습된 데이터(예: 2024년까지)까지만 알고 있습니다. 뉴스룸의 실시간 기사 DB, 오늘의 주가, 현재 날씨를 모릅니다.

Function Calling의 핵심 개념.

중요: "Function Calling"이라는 이름은 오해를 부릅니다. LLM이 직접 함수를 실행하는 게 아닙니다.

실제 프로세스는 이렇습니다:

Step 1: 인간이 함수 정의를 LLM에 설명.

API 요청에 "tools" 파라미터를 포함시켜, "search_articles"라는 함수가 존재하고, 어떤 파라미터를 받는지 설명합니다.

Step 2: LLM이 구조화된 응답 생성.

LLM은 함수를 실행하지 않고, "이 함수를 search_articles 파라미터 {'keyword': '경제', 'category': '금융'} 로 호출해야겠다"는 구조화된 응답을 생성합니다.

Step 3: 인간(또는 프로그램)이 함수 실행.

LLM의 응답을 파싱하고, 실제로 데이터베이스를 조회하거나 API를 호출합니다. 결과를 LLM에 다시 피드백합니다.

비유: LLM은 간호사가 아니라 의사입니다. 환자를 진찰하고 "혈액검사를 해주세요"라는 지시를 내리지만, 직접 혈액을 채취하지는 않습니다. 간호사(프로그래머)가 실행하고 결과를 의사에게 전달합니다.

    messages=messages

print(final_response.choices[0].message.content)

Function Calling의 5가지 활용 사례.

1. 이메일 발송 자동화.

사용자: "어제 편집 완료한 기자들에게 감사 이메일 보내줘." → LLM이 "send_email" 함수를 호출 → 자동으로 메일 전송.

2. 일정 관리.

사용자: "오늘부터 1주일 내 마감이 있는 기사 찾아줘" → LLM이 "get_articles_with_deadline" 함수를 호출 → DB에서 필터링된 결과 반환.

3. 데이터베이스 쿼리.

복잡한 SQL을 직접 쓸 필요 없습니다. 자연어로 "지난달 서울 지역 범죄 기사 중 댓글이 1000개 이상인 것만"이라고 하면 LLM이 적절한 쿼리 파라미터를 생성합니다.

4. 외부 API 연동.

뉴스룸은 각종 데이터 API(날씨, 주가, 환율, 부동산 시세 등)를 호출해야 합니다. Function Calling으로 "어제의 달러 환율이 뭐야?"라고 물으면 자동으로 환율 API를 호출합니다.

5. 데이터 검증 및 포맷팅.

"전국 선거 투표율을 지역별로 정렬해줘" → LLM이 "format_election_data" 함수를 호출 → 일관된 JSON 형태로 반환.

  • 3단계 프로세스: API 요청 → 응답 파싱 → 함수 실행 → 결과 피드백.
  • 뉴스룸에서는 선거 데이터, 주가, 환율, 댓글 분석 등 다양한 외부 데이터를 자동으로 조회할 수 있습니다.

퀴즈.

OpenAI API 주요 파라미터 정리.

기본 파라미터:

  • model: 사용할 모델 ("gpt-4o-mini", "gpt-4", "text-embedding-3-small" 등)
  • messages: 대화 히스토리 배열. 각 메시지는 {"role": "user/assistant/system", "content": "..."}
  • temperature: 0~2 범위. 0은 확정적, 1은 창의적. (기본값: 1)
  • max_tokens: 응답 길이 제한. (기본값: 무제한)
  • top_p: 0.1~1 범위의 핵심 샘플링. temperature와 함께 사용하지 말 것.

고급 파라미터:

  • system: (messages[0].role == "system") AI의 성격과 역할 지시.
  • tools: Function Calling용 함수 정의 배열.
  • tool_choice: "auto" (자동), "required" (필수), {"type": "function", "function": {"name": "..."}} (특정 함수 강제).
  • response_format: {"type": "json_object"} (JSON 응답 강제)

Image 생성 (DALL-E):

  • prompt: 이미지 설명 (1000자 이상 권장)
  • n: 생성할 이미지 개수 (기본: 1)
  • size: "1024x1024", "1792x1024", "1024x1792"
  • quality: "standard" (빠름), "hd" (디테일)

Embedding:

  • input: 벡터화할 텍스트 (문자열 또는 배열)
  • model: "text-embedding-3-small" (저비용), "text-embedding-3-large" (고정확)

Vision (이미지 분석):

  • messages[].content: 텍스트와 이미지 혼합 가능
  • image_url: {"url": "https://..."} 또는 base64 인코딩
  • detail: "low" (빠름), "high" (정밀함)

Q: API 비용은 얼마나 드나요?

A: 모델마다 다릅니다. gpt-4o-mini는 저렴한 편입니다. OpenAI 요금표를 참고하세요. 프리 크레딧으로 테스트할 수 있습니다.

Q: 한국어를 잘 처리할 수 있나요?

A: 네, gpt-4o-mini는 한국어 이해와 생성이 우수합니다. 하지만 영어보다는 비용이 약간 높을 수 있습니다.
