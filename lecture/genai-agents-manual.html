AI 에이전트란?

에이전트는 AI의 자율성을 한 단계 높인 시스템입니다. 목표를 받으면 스스로 계획을 세우고, 필요한 도구를 선택해 사용하고, 중간 결과를 확인한 후 다음 행동을 결정합니다.

프롬프트와 에이전트의 본질적 차이.

프롬프트 (Prompt)

동작 원리: 사용자가 질문 → LLM이 학습 데이터 기반으로 즉시 답변.

예시: "2024년 한국 경제 성장률은?"

문제점: 학습 데이터의 최신 정보로만 답변. 최신 뉴스나 실시간 데이터 불가능.

에이전트 (Agent)

동작 원리: 목표 수신 → 계획 수립 → 도구 선택 → 실행 → 결과 확인 → 다음 행동 결정 (반복).

예시: "2025년 한국 경제 전망을 조사하고 요약해줘."

장점: 웹 검색, DB 조회, 계산 등 외부 도구를 자동으로 사용. 최신 정보 반영 가능.

뉴스룸 비유로 이해하기.

프롬프트 기반 뉴스룸: 기자가 AI에게 "이 사건의 배경을 설명해줘"라고 물으면, AI가 아는 것만 답변. 최신 정보나 추가 자료 없음.

에이전트 기반 뉴스룸: 기자가 AI에게 "이 사건을 취재해줘"라고 지시하면, AI가 자동으로 관련 기사 검색 → 등장인물 조사 → 과거 기록 확인 → 통계 데이터 수집 → 요약 작성을 수행. 기자는 완성된 배경자료를 받음.

에이전트의 핵심 구성 요소.

모든 에이전트는 네 가지 핵심 요소로 구성됩니다.

1. LLM (두뇌: 판단과 생성)

LLM은 에이전트의 중추입니다. 현재 상태를 보고 "이제 어떤 도구를 써야 하나?" 또는 "답변이 충분한가?"를 판단합니다.

from langchain_openai import ChatOpenAI

# GPT-4o mini 모델로 에이전트의 두뇌 만들기
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0,  # 결정적 응답 (뉴스 작성 시 중요)
    api_key="your-openai-api-key"
)

# LLM 호출 테스트
response = llm.invoke("한국 뉴스룸에서 AI 에이전트의 역할은?")
print(response.content)

2. 도구 (Tools: 외부 액션)

    return f"'{query}' 검색 결과를 반환합니다."

# 계산 도구
@tool
def calculate(expression: str) -> str:
    """수학 계산을 수행합니다."""
    return str(eval(expression))

LangGraph 소개.

LangGraph는 그래프 기반의 워크플로우 프레임워크입니다. 복잡한 에이전트 로직을 시각적으로 설계할 수 있습니다.

핵심 개념: 노드와 엣지.

노드 (Node): 작업을 수행하는 단계. 예: "웹 검색하기", "결과 분석하기", "요약 작성하기".

엣지 (Edge): 노드 간의 흐름. 한 노드가 끝나면 다음 노드로 이동.

조건부 분기 (Conditional Edge): 중간 결과에 따라 다른 경로로 이동. 예: 검색 결과가 충분하면 요약으로 가고, 부족하면 다시 검색.
Python 
# LangGraph의 기본 흐름도
# [입력] → [검색 노드] → [분석 노드] → [요약 노드] → [출력]
#            ↓          ↓
#         [도구 호출] [조건 확인: 충분한가?]
#            ↓          ↓
#         [결과]   아니오 → 다시 검색

LangGraph 기본 구조.

간단한 에이전트부터 시작합니다. 사용자가 작업을 입력하면, 에이전트가 처리하고 결과를 반환합니다.

Python 
from langgraph.graph import StateGraph, END
from typing import TypedDict
from langchain_openai import ChatOpenAI

# Step 1: 상태 정의
class AgentState(TypedDict):
    task: str
    result: str

# Step 2: LLM 초기화
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# Step 3: 노드 함수 정의
def process_node(state: AgentState):
    """작업을 처리하는 노드"""
    result = llm.invoke(state["task"])
    return {"result": result.content}

# Step 4: 그래프 생성
graph = StateGraph(AgentState)
graph.add_node("process", process_node)
graph.set_entry_point("process")
graph.add_edge("process", END)

# Step 5: 컴파일
app = graph.compile()

# Step 6: 실행
output = app.invoke({
    "task": "서울시 예산안의 핵심 쟁점 3가지를 정리해줘."
})

print("결과:", output["result"])

더 복잡한 워크플로우 (조건부 분기).

Python 
# 상태 확장
class ResearchState(TypedDict):
    topic: str
    search_count: int
    articles: str
    is_sufficient: bool
    summary: str

# 노드 1: 검색
def search_node(state: ResearchState):
    """뉴스 검색"""
    return {
        "articles": "검색 결과 3개...",
        "search_count": state["search_count"] + 1
    }

# 노드 2: 결과 평가
def evaluate_node(state: ResearchState):
    """충분한 정보를 수집했는가?"""
    is_sufficient = len(state["articles"]) > 100
    return {"is_sufficient": is_sufficient}

# 노드 3: 요약
def summary_node(state: ResearchState):
    """최종 요약"""
    summary = llm.invoke(
        f"다음 기사들을 요약해줘: {state['articles']}"
    )
    return {"summary": summary.content}

# 조건부 분기 함수
def should_summarize(state: ResearchState):
    """충분한 정보가 모였으면 요약, 아니면 다시 검색"""
    if state["is_sufficient"] or state["search_count"] >= 3:
        return "summary"
    else:
        return "search"

# 그래프 구성
graph = StateGraph(ResearchState)
graph.add_node("search", search_node)
graph.add_node("evaluate", evaluate_node)
graph.add_node("summary", summary_node)

graph.set_entry_point("search")
graph.add_edge("search", "evaluate")
# 조건부 분기: evaluate 다음에 should_summarize 함수로 판단
graph.add_conditional_edges("evaluate", should_summarize)
graph.add_edge("summary", END)

app = graph.compile()

class NewsState(TypedDict):
    # 입력
    event: str  # 뉴스 이벤트
    # 중간 단계
    background: str  # 배경 정보
    key_points: str  # 핵심 포인트
    related_news: str  # 관련 뉴스
    # 최종 결과
    article: str  # 완성된 기사

def research_background(state: NewsState):
    """배경 정보 수집"""
    background = "... (배경 정보 조사) "
    return {"background": background}  # background만 업데이트

def extract_key_points(state: NewsState):
    """핵심 포인트 추출"""
    # 이전 노드가 설정한 background를 읽을 수 있음
    key_points = llm.invoke(
        f"다음 배경에서 핵심 3가지를 추출해줘: {state['background']}"
    )
    return {"key_points": key_points.content}

def find_related_news(state: NewsState):
    """관련 뉴스 검색"""
    related = "... (관련 뉴스 검색) "
    return {"related_news": related}

def write_article(state: NewsState):
    """최종 기사 작성"""
    # 모든 이전 정보에 접근
    article = llm.invoke(
        f"""
주제: {state['event']}
배경: {state['background']}
핵심: {state['key_points']}
관련뉴스: {state['related_news']}

위 정보를 바탕으로 기사를 작성해줘."""
    )
    return {"article": article.content}

        f"'{state['topic']}'에 대한 기사 구성안 (제목, 리드, 본문 3단락, 결론)을 만들어줘."
    )

from langgraph.graph import StateGraph, END
from typing import TypedDict
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 상태 정의: 메시지 리스트를 중심으로
class ChatState(TypedDict):
    messages: list  # 대화 기록
    context: str  # 추가 맥락 (선택사항)

# 초기 시스템 메시지
SYSTEM_PROMPT = """당신은 탐사보도 전문 취재 도우미입니다.
- 기자의 질문에 배경 정보를 제공하세요.
- 정확한 정보만 제시하세요.
- 필요시 추가 조사가 필요한 부분을 명시하세요.
- 한국어로 답변하세요."""

# 대화 노드
def chat_node(state: ChatState):
    """사용자 질문에 대답"""
    # 시스템 메시지 + 대화 기록으로 LLM 호출
    system_message = SystemMessage(content=SYSTEM_PROMPT)
    all_messages = [system_message] + state["messages"]

    response = llm.invoke(all_messages)

    # 새로운 AI 메시지를 메모리에 추가
    new_messages = state["messages"] + [AIMessage(content=response.content)]

    return {"messages": new_messages}

# 그래프 구성
graph = StateGraph(ChatState)
graph.add_node("chat", chat_node)
graph.set_entry_point("chat")
graph.add_edge("chat", END)

agent = graph.compile()

# 사용 예시
# 첫 번째 질문
state1 = agent.invoke({
    "messages": [HumanMessage(content="삼성 반도체 뉴스를 찾아줄래?")],
    "context": ""
})

# 두 번째 질문 (메모리 유지)

    "messages": state1["messages"] + [HumanMessage(content="그 중 최신 뉴스 3개만 정리해줄래?")],
    "context": ""
})

from langgraph.graph import StateGraph, END

class RoutingChatState(TypedDict):
    messages: list
    query_type: str  # "factcheck", "summarize", "research", "general"

# 질문 유형 분류
def classify_query(state: RoutingChatState):
    """마지막 사용자 메시지의 유형 분류"""
    last_msg = state["messages"][-1].content.lower()

    if "팩트" in last_msg or "확인" in last_msg:
        query_type = "factcheck"
    elif "요약" in last_msg:
        query_type = "summarize"
    elif "조사" in last_msg or "찾" in last_msg:
        query_type = "research"
    else:
        query_type = "general"

    return {"query_type": query_type}

# 각 유형별 응답 노드
def factcheck_node(state: RoutingChatState):
    system = SystemMessage(content=FACTCHECKER_PROMPT)
    response = llm.invoke([system] + state["messages"])
    new_messages = state["messages"] + [AIMessage(content=response.content)]
    return {"messages": new_messages}

def summarize_node(state: RoutingChatState):
    system = SystemMessage(content="주어진 텍스트를 명확하고 간결하게 요약해줘.")
    response = llm.invoke([system] + state["messages"])
    new_messages = state["messages"] + [AIMessage(content=response.content)]
    return {"messages": new_messages}

def research_node(state: RoutingChatState):
    system = SystemMessage(content=REPORTER_PROMPT)
    response = llm.invoke([system] + state["messages"])
    new_messages = state["messages"] + [AIMessage(content=response.content)]
    return {"messages": new_messages}

def general_node(state: RoutingChatState):
    system = SystemMessage(content="사용자의 질문에 도움이 되도록 답변해줘.")
    response = llm.invoke([system] + state["messages"])
    new_messages = state["messages"] + [AIMessage(content=response.content)]
    return {"messages": new_messages}

# 라우팅 함수
def route_by_query_type(state: RoutingChatState):
    return state["query_type"]

# 그래프 구성
graph = StateGraph(RoutingChatState)
graph.add_node("classify", classify_query)
graph.add_node("factcheck", factcheck_node)
graph.add_node("summarize", summarize_node)
graph.add_node("research", research_node)
graph.add_node("general", general_node)

graph.set_entry_point("classify")
graph.add_conditional_edges("classify", route_by_query_type, {
    "factcheck": "factcheck",
    "summarize": "summarize",
    "research": "research",
    "general": "general"
})
graph.add_edge("factcheck", END)
graph.add_edge("summarize", END)
graph.add_edge("research", END)
graph.add_edge("general", END)

agent = graph.compile()

        "timestamp": str(datetime.now()),
        "messages": [

                "content": msg.content
            }
            for msg in messages
        ]
    }

        json.dump(data, f, ensure_ascii=False, indent=2)

도구 사용 에이전트.

45분 중급 도구 웹검색

에이전트가 도구를 사용하는 원리.

LLM은 자체로는 웹을 검색하거나 계산을 할 수 없습니다. 도구를 사용함으로써 외부 정보와 계산 능력을 얻습니다. 에이전트의 핵심은 "언제 어떤 도구를 써야 하는가"를 판단하는 것입니다.

도구 사용의 흐름.

Step 1: 사용자가 작업을 요청. 예: "올해 최저임금 인상률을 검색하고, 월급으로 환산해줘."

Step 2: LLM이 작업을 분석하고 필요한 도구 판단. 예: "search_news 도구를 먼저 사용해야 해."

Step 3: 도구 호출. 예: search_news(query="2024 최저임금 인상률")

Step 4: 도구 실행 결과 받음. 예: "최저임금이 10,410원으로 인상되었습니다."

Step 5: 결과를 LLM에 제공하고 다음 도구 판단. 예: "이제 월급 계산 도구를 사용해야 해."

Step 6: 최종 결과 도출. 예: "일일급 83,280원, 월급(240시간 기준) 2,499,600원"

도구 정의하기.

도구는 Python 함수에 @tool 데코레이터를 붙여서 정의합니다. 도구는 명확한 입력과 출력을 가져야 합니다.

기본 도구 정의.

Python 
from langchain.tools import tool

# 예제 1: 웹 검색 도구

from typing import TypedDict

class BackgroundResearchState(TypedDict):
    target: str  # 조사 대상 (인물, 회사, 사건)
    search_results: str  # 검색 결과
    analysis: str  # 분석 결과

def search_step(state: BackgroundResearchState):
    """배경 정보 검색"""
    target = state["target"]
    results = search_korean_news(target, max_results=10)
    return {"search_results": results}

def analyze_step(state: BackgroundResearchState):
    """검색 결과 분석"""
    analysis_prompt = f"""
다음 검색 결과를 분석해줘:
주제: {state['target']}

검색 결과:
{state['search_results']}

분석 내용:
1. 주요 사건/뉴스 (최근순)
2. 핵심 인물/단체
3. 배경 및 원인
4. 영향 및 결과
5. 향후 전망
"""

    response = llm.invoke(analysis_prompt)
    return {"analysis": response.content}

# 그래프 구성
graph = StateGraph(BackgroundResearchState)
graph.add_node("search", search_step)
graph.add_node("analyze", analyze_step)

graph.set_entry_point("search")
graph.add_edge("search", "analyze")
graph.add_edge("analyze", END)

research_agent = graph.compile()

# 사용

    "target": "이재용 삼성전자 회장",
    "search_results": "",
    "analysis": ""
})

def factcheck_node(state: JournalismState):
    from duckduckgo_search import DDGS

    # 기사에서 검증 가능한 주장 추출
    claims_prompt = f"""기사에서 사실 여부를 검증할 수 있는 주장을 추출하세요.
<article>{state['article']}</article>"""
    claims = llm.invoke(claims_prompt).content

    # 각 주장을 웹 검색으로 검증
    search = DDGS()
    results = search.text(claims[:100], region="kr-kr", max_results=3)

    verify_prompt = f"""주장: {claims}
검색 결과: {results}

각 주장의 사실 여부를 [확인됨/미확인/반박됨]으로 판정하세요."""
    verification = llm.invoke(verify_prompt)
    return {"fact_check": verification.content}

def summarize_articles(state: NewsTLDRState):
    """각 기사를 2문장으로 요약."""
    summaries = []

    for article in state["articles"]:
        content = article.get("content") or article.get("description", "")

        if not content:
            continue

        prompt = f"""다음 기사를 2문장으로 요약하세요.
        제목: {article['title']}
        본문: {content[:500]}"""

        summary = llm.invoke(prompt).content
        summaries.append({
            "title": article["title"],
            "summary": summary,
            "url": article["url"],
            "source": article["source"].get("name", "알 수 없음")
        })

    return {"summaries": summaries}

def generate_briefing(state: NewsTLDRState):

import time
from requests.exceptions import RequestException

def fetch_with_retry(query, api_key, max_retries=3):
    """최대 3회 재시도하며 뉴스 수집."""
    for attempt in range(max_retries):
        try:
            articles = fetch_news(query, api_key)
            if articles:
                return articles
        except RequestException as e:
            print(f"시도 {attempt + 1} 실패: {e}")
            if attempt < max_retries - 1:
                # 지수 백오프: 1초, 2초, 4초
                wait_time = 2 ** attempt
                print(f"{wait_time}초 후 재시도...")
                time.sleep(wait_time)

    return []

        print(f"에러 발생: {e}")
        raise

목적: 기사를 입력받아 요약, 팩트체크, 톤 분석 수행.
