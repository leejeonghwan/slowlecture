1-2. 첫 번째 API 호출

아래 코드가 기본 구조입니다. messages 리스트에 시스템 프롬프트와 사용자 질문을 담아 보냅니다.

Python
from openai import OpenAI

client = OpenAI(api_key="여기에_API키_입력")

messages = [
    {"role": "system", "content": "당신은 유능한 비서입니다."},
    {"role": "user",   "content": "김치를 담글 때 소금과 배추의 비율은?"}
]

completion = client.chat.completions.create(
    messages=messages,
    model='gpt-4o-mini'
)

print(completion.choices[0].message.content)
핵심 개념: system / user / assistant 역할 ▶

system — 모델의 행동 방식을 지정하는 시스템 프롬프트입니다. "당신은 유능한 비서입니다"와 같은 지시를 넣습니다.

user — 사용자의 질문이나 요청입니다.

assistant — 모델의 이전 응답입니다. 대화 맥락을 이어가고 싶을 때 이전 응답을 포함시킵니다.

1-4. 온도(Temperature) 설정.

온도는 모델 응답의 "창의성" 수준을 조절하는 파라미터입니다. 통계적으로 말하면, 다음에 올 단어를 선택할 때 얼마나 덜 가능성 있는 단어도 허용할지를 결정합니다.

온도특성취재 활용
0.0거의 동일한 결과 재현분류, 데이터 추출 (추천)
0.7적당한 변화 (기본값)일반적인 질의응답
2.0매우 창의적/예측 불가브레인스토밍용

1-6. 대량 처리 — pandas와 함께.

스프레드시트의 모든 행에 대해 분류를 자동 실행하는 패턴입니다. 실제 취재에서 수백~수천 건의 문서를 처리할 때 사용합니다.

CSV/엑셀 데이터
→
프롬프트 템플릿
→

→

Python — 대량 분류 패턴
import pandas as pd
from tqdm.notebook import tqdm

df = pd.read_csv("my-data.csv")

prompt_template = """
다음 법안을 환경, 보건, 이민, 세금, 기타 중 하나로 분류하세요.
카테고리 이름만 답하세요.

법안 제목: {text}
"""

def llm_request(row):
    prompt = prompt_template.format(text=row['title'])
    messages = [
        {"role": "system", "content": "당신은 입법 보좌관입니다."},
        {"role": "user",   "content": prompt}
    ]
    completion = client.chat.completions.create(
        messages=messages, model="gpt-4o-mini", temperature=0
    )
    return completion.choices[0].message.content

# 아직 처리 안 된 행만 처리 (중단 후 재실행 대비)
if 'answer' not in df.columns:
    df['answer'] = None

mask = df['answer'].isna()
for row in tqdm(df[mask].itertuples(), total=mask.sum()):
    try:
        result = llm_request(row)
        df.loc[row.Index, 'answer'] = result
    except Exception as e:
        print(f"오류 발생 (행 {row.Index}): {e}")
✅ 실전 팁: 중단 복구 패턴.
위 코드에서 df['answer'].isna()로 미처리 행만 필터링합니다. API 호출 중 오류가 나도 이미 처리한 데이터는 보존되므로, 코드를 다시 실행하면 남은 행부터 이어서 처리됩니다.

1-8. 구조화 출력 (Structured Outputs)

LLM은 규칙을 어기고 예상 밖의 형식으로 응답할 수 있습니다. 구조화 출력을 사용하면 원하는 형태로 정확하게 응답을 받을 수 있습니다.

Python — Pydantic으로 출력 구조 정의
from pydantic import BaseModel, Field
from typing import Literal

class Comment(BaseModel):
    """댓글 데이터 모델"""
    name: str = Field(description="작성자 이름")
    food_item: str = Field(description="논의 중인 음식, 영문")
    email: str = Field(description="작성자 이메일")
    emotion: Literal["positive", "negative", "neutral"] = Field(
        description="댓글의 감성"
    )
Python — 구조화 출력으로 API 호출
completion = client.beta.chat.completions.parse(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "관련 정보를 추출하세요."},
        {"role": "user",   "content": email_text},
    ],
    response_format=Comment,  # 위에서 정의한 Pydantic 모델
)

result = completion.choices[0].message.parsed
print(result.model_dump())
# {'name': '홍길동', 'food_item': 'broccoli', 'email': '...', 'emotion': 'negative'}

다국어 기사를 분석할 때 Pydantic 모델을 이렇게 정의할 수 있습니다: headline(제목), outlet_name(매체명), summary(요약), language(언어코드), sources(취재원 목록), category(분야). 수백 건의 기사를 한 번에 구조화된 스프레드시트로 변환할 수 있습니다.

2-1. Whisper — 음성을 텍스트로.

Whisper는 OpenAI가 공개한 음성 전사(speech-to-text) 모델입니다. GPT와 달리 직접 다운로드해서 로컬에서 실행할 수 있습니다. 인터뷰 녹음, 회의록, 브리핑 영상 등을 텍스트로 변환하는 데 활용됩니다.

Python — Whisper 기본 사용법
import whisper

# 모델 크기: tiny, base, small, medium, large, turbo
model = whisper.load_model("medium")  # 한국어는 medium 이상 권장

result = model.transcribe("interview.mp3")
print(result["text"])
모델크기속도정확도한국어
tiny39MB매우 빠름낮음❌ 비추천
small244MB빠름보통⚠️ 기초적
medium769MB보통좋음✅ 사용 가능
turbo809MB빠름좋음✅ 추천
🚨 Whisper의 치명적 문제: 환각(Hallucination)
AP통신 보도에 따르면 Whisper는 심각한 오류를 만들 수 있습니다. 특히 무음 구간에서 존재하지 않는 문장을 생성하는 경우가 보고되었습니다. 반드시 결과를 사람이 검수해야 합니다. 전사 결과를 그대로 보도에 사용하지 마세요.

2-2. WhisperX — 개선된 전사.

WhisperX는 Whisper에 음성 활동 감지(VAD), 화자 분리, 타임스탬프 정렬 등을 추가한 라이브러리입니다. 무음 구간의 환각 문제를 크게 줄여줍니다.

Python — WhisperX 사용법
import whisperx, torch

device = "cuda" if torch.cuda.is_available() else "cpu"

# 1. 전사
model = whisperx.load_model("turbo", device)
audio = whisperx.load_audio("interview.mp3")
result = model.transcribe(audio, batch_size=4)

# 2. 타임스탬프 정렬
model_a, metadata = whisperx.load_align_model(
    language_code=result["language"], device=device
)
result = whisperx.align(
    result["segments"], model_a, metadata, audio, device
)

# 3. 결과를 DataFrame으로
import pandas as pd
df = pd.json_normalize(result['segments'])
text = "\n".join(df['text'])
print(text[:500])

import yt_dlp

url = "https://www.youtube.com/watch?v=VIDEO_ID"

ydl_opts = {
    'format': 'bestaudio/best',
    'outtmpl': '%(title)s.%(ext)s',
    'postprocessors': [
        {'key': 'FFmpegExtractAudio', 'preferredcodec': 'mp3'},
    ],
}

with yt_dlp.YoutubeDL(ydl_opts) as ydl:
    ydl.download([url])

양자화(Quantization) 이해하기 ▶

거대 모델을 일반 컴퓨터에서 실행하려면 양자화가 필요합니다. 모델 내부의 숫자를 반올림해서 크기를 줄이는 기법입니다.

q8(고품질) → q6 → q4(추천) → q2(저품질) 순으로 작아집니다. 보통 q4면 충분한 성능을 유지합니다. 36B 모델의 q4 버전이 7B 원본 모델보다 더 나은 경우가 많습니다.

Hugging Face는 AI 모델의 YouTube와 같습니다. 수만 개의 모델이 공개되어 있으며, 특정 작업에 특화된 모델을 찾아 쓸 수 있습니다.

임베딩과 문서 검색.

중급 60분
텍스트를 숫자로 변환(임베딩), 문서 유사도 비교, RAG 기반 문서 Q&A

3-1. 임베딩이란?

컴퓨터는 숫자만 이해합니다. 임베딩(embedding)은 텍스트를 숫자 벡터(리스트)로 변환하는 기술입니다. 의미가 비슷한 문장은 비슷한 숫자 패턴을 갖게 됩니다.

"고양이가 생선을 먹었다"
→
임베딩 모델
→
[0.23, -0.41, 0.87, ...]
Python — 문장 임베딩 생성
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('sentence-transformers/all-MiniLM-L6-v2')

sentences = ["고양이가 생선을 먹었다"]
embeddings = model.encode(sentences)

print(embeddings[0][:10])  # 처음 10개 숫자만 출력

3-2. 문서 유사도 비교.

임베딩을 이용하면 문장 간 의미적 유사도를 계산할 수 있습니다. 코사인 유사도(cosine similarity)가 1에 가까울수록 의미가 비슷합니다.

Python — 유사도 행렬 생성
from sklearn.metrics.pairwise import cosine_similarity
import pandas as pd

sentences = [
    "아파트 가격이 급등했다",
    "부동산 시장이 과열되고 있다",
    "오늘 야구 경기에서 홈런이 나왔다",
    "Цены на недвижимость выросли",  # 러시아어: 부동산 가격 상승
]

embeddings = model.encode(sentences)
similarities = cosine_similarity(embeddings)

pd.DataFrame(similarities,
    index=sentences, columns=sentences
).style.background_gradient(axis=None)
✅ 다국어 임베딩.
all-MiniLM-L6-v2는 영어 전용입니다. 한국어를 포함한 다국어 문서를 비교하려면 distiluse-base-multilingual-cased-v2나 BAAI/bge-m3를 사용하세요. ICIJ의 루안다 리크스 탐사보도에서도 300GB 이상의 다국어 데이터를 임베딩으로 탐색했습니다.

3-3. RAG — 문서 기반 질의응답

RAG(Retrieval-Augmented Generation)는 LLM이 보유 문서에서 관련 내용을 찾아 답변하는 방식입니다. LlamaIndex를 사용하면 간단하게 구현할 수 있습니다.

질문 입력
→
관련 문서 검색
(임베딩 유사도)
→
LLM에 질문+문서 전달
→
근거 기반 답변
Python — LlamaIndex로 문서 Q&A
import os
os.environ['OPENAI_API_KEY'] = '여기에_API키'

from llama_index.core import VectorStoreIndex, SimpleDirectoryReader

# 1. 문서 폴더 로드 (PDF, TXT 등)
documents = SimpleDirectoryReader("documents").load_data()
index = VectorStoreIndex.from_documents(documents)

# 2. 질의
query_engine = index.as_query_engine(similarity_top_k=5)
response = query_engine.query("직원의 팁을 가져가도 되나요?")
print(response)

# 3. 출처 확인
for node in response.source_nodes:
    print(f"파일: {node.node.metadata}")
    print(f"관련도: {node.score:.3f}")
⚠️ RAG의 한계
뉴욕시가 만든 AI 챗봇이 RAG를 사용했지만 사업자에게 법을 어기라고 조언한 사례가 있습니다 (The Markup 보도). RAG 결과를 그대로 독자에게 서비스하지 말고, 취재의 보조 도구로만 활용하세요.

Gradio로 앱 만들기.

입문 60분
비개발자도 사용할 수 있는 웹 인터페이스를 Python 코드 몇 줄로 만들기

왜 앱을 만들어야 할까?

기술팀만 AI를 테스트하면 시각이 편향됩니다. 뉴스룸의 비기술 동료들이 직접 도구를 시험할 수 있게 하면, 맹점을 발견하고 더 나은 결과를 얻을 수 있습니다. Gradio는 Python 함수를 웹 인터페이스로 즉시 변환해 줍니다.

4-1. Gradio 기본 패턴.

모든 Gradio 앱은 동일한 3단계 구조를 따릅니다.

1

함수 정의.

입력을 받아 결과를 반환하는 Python 함수를 작성합니다.

2

인터페이스 설정.

gr.Interface(fn=함수, inputs=입력타입, outputs=출력타입)으로 UI를 구성합니다.

3

실행

iface.launch()로 브라우저에서 바로 접속 가능한 웹앱을 띄웁니다.

Python — Gradio 기본 예제
import gradio as gr

def greet(name):
    return "안녕하세요, " + name + "님!"

iface = gr.Interface(fn=greet, inputs="text", outputs="text")
iface.launch()  # 브라우저에서 http://127.0.0.1:7860 접속

4-2. 실전 예제 모음.

녹음 파일을 업로드하면 텍스트로 전사해주는 앱입니다. 모델 크기를 달리 해서 결과를 비교할 수도 있습니다.

Python — 음성 전사 앱 (모델 비교)
import gradio as gr
import whisper

tiny_model = whisper.load_model("tiny")
med_model = whisper.load_model("medium")

def transcribe(audio_file):
    result_tiny = tiny_model.transcribe(audio_file)
    result_med = med_model.transcribe(audio_file)
    return result_tiny['text'], result_med['text']

iface = gr.Interface(
    fn=transcribe,
    inputs=gr.Audio(type="filepath", label="녹음 파일 업로드"),
    outputs=[
        gr.Text(label="Tiny 모델 결과"),
        gr.Text(label="Medium 모델 결과"),
    ],
    allow_flagging='never'
)
iface.launch()

텍스트를 입력하면 미리 정한 카테고리로 분류해주는 앱입니다. gr.Label 컴포넌트로 확률 분포를 시각화합니다.

Python — 분류 앱
import gradio as gr
from transformers import pipeline

classifier = pipeline("zero-shot-classification",
                      model="facebook/bart-large-mnli")

def classify(text):
    labels = ['정치', '경제', '사회', '국제', '스포츠', '문화']
    result = classifier(text, labels)
    return dict(zip(result['labels'], result['scores']))

iface = gr.Interface(fn=classify, inputs="text", outputs="label")
iface.launch()

기사 원고를 붙여넣으면 AP 스타일가이드 기준으로 교정 제안을 해주는 앱입니다.

Python — 기사 교정 앱 (핵심부)
def get_feedback(story_text):
    prompt = f"""
    아래 기사의 문체와 맞춤법을 교정해주세요.
    한국어 기사 작성 기준을 따르고,
    구체적인 수정 사항을 제시하세요.

    기사 원문:
    {story_text}
    """

    messages = [
        {"role": "system", "content": "당신은 숙련된 교열 기자입니다."},
        {"role": "user",   "content": prompt}
    ]

    completion = client.chat.completions.create(
        messages=messages, model="gpt-4o-mini", temperature=0
    )
    return completion.choices[0].message.content

iface = gr.Interface(
    fn=get_feedback,
    inputs=gr.Textbox(lines=15, label="기사 원문"),
    outputs=gr.Textbox(label="교정 제안"),
    allow_flagging='never'
)
iface.launch()

5-2. 프로젝트 아이디어.

아래는 워크숍에서 제안된 프로젝트들입니다. 한국 뉴스룸에 맞게 변형해서 시도해 보세요.

PDF 문서 일괄 처리기 ▶

PDF를 드래그앤드롭하면 각 문서의 내용이 스프레드시트 한 행이 됩니다. OCR 기능을 추가하면 스캔된 문서도 처리 가능합니다.

활용 예: 국정감사 자료, 법원 판결문, 감사보고서 등을 일괄 분석할 때

지방의회 회의록 자동 요약기 ▶

YouTube URL(의회 영상)을 입력하고 관심 키워드를 지정하면, 해당 부분만 발췌·요약해줍니다. Whisper + RAG + Gradio 조합입니다.

활용 예: 지자체 의회, 시도교육청 회의 모니터링

문서 시맨틱 검색 도구 ▶

대량의 문서(인터뷰 노트, 보도자료 등)에서 의미 기반 검색을 수행합니다. 정확한 키워드를 모를 때도 관련 내용을 찾을 수 있습니다.

활용 예: 수십 건의 인터뷰 노트에서 특정 주제 언급 부분 일괄 검색

뉴스 퀴즈 생성기 ▶

질문·답·이미지를 입력하면 인터랙티브 뉴스 퀴즈 HTML이 생성됩니다. CMS에 바로 삽입 가능합니다.
