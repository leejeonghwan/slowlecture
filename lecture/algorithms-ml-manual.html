M1: Python 데이터 구조 복습

초급 30분

개요

Python의 기본 데이터 구조를 저널리즘 실무 예제를 통해 복습합니다. 리스트, 튜플, 딕셔너리, 집합 각각이 언제 어떻게 쓰이는지 알아봅시다.

1. 리스트 (List): 순서가 있고 변경 가능

리스트는 여러 항목을 순서대로 저장할 때 사용합니다. 저널리즘에서는 기사 제목, 취재원 명단, 보도 시간 등을 저장하는 데 자주 씁니다.

Python 
# 기사 제목 목록
articles = [
    "국회, 신청사 이전 결정",
    "미세먼지 경보 발령",
    "삼성전자 4분기 실적 발표"
]

# 리스트의 첫 번째 항목 접근 (인덱싱)
print(articles[0])  # "국회, 신청사 이전 결정"

# 리스트에 새 기사 추가
articles.append("날씨 내일 맑음")

# 리스트 길이 확인
print(len(articles))  # 4

리스트 컴프리헨션

리스트 컴프리헨션은 반복문과 조건을 한 줄로 작성하는 Python의 강력한 기능입니다:

sources = [
    "김민준 국회의원",

# 10자 이상인 이름만 필터링
long_names = [name for name in sources if len(name) >= 10]

3. 딕셔너리 (Dictionary): 키-값 쌍

딕셔너리는 키로 값을 찾는 구조입니다. 기사의 메타데이터(제목, 기자, 날짜, 조회수)를 저장하기에 완벽합니다.

Python 
# 기사 메타데이터
article = {
    "제목": "국회, 신청사 이전 결정",
    "기자": "김민준",
    "날짜": "2026-03-28",
    "조회수": 125000,
    "카테고리": "정치"
}

# 키로 값 접근
print(article["제목"])  # "국회, 신청사 이전 결정"

# get() 메서드: 키가 없으면 기본값 반환
print(article.get("작성자", "정보 없음"))  # "정보 없음"

# 새로운 정보 추가
article["편집자"] = "이영희"

# 모든 키 확인
print(article.keys())
팁: 딕셔너리의 items() 메서드를 사용하면 키와 값을 동시에 순회할 수 있습니다.

# 딕셔너리의 모든 항목 순회
for key, value in article.items():

# 결과:

# 기자: 김민준
# 날짜: 2026-03-28

4. 집합 (Set): 유일한 항목들의 모음

집합은 중복을 자동으로 제거합니다. 기사의 모든 태그를 수집할 때나 중복 제거가 필요할 때 유용합니다.

Python 
# 여러 기사에서 수집한 태그들 (중복 있음)
tags = ["정치", "경제", "정치", "사회", "경제", "국제"]

# 중복을 제거한 유일한 태그들
unique_tags = set(tags)
print(unique_tags)  # {'정치', '경제', '사회', '국제'}

# 새 태그 추가
unique_tags.add("과학")

# 태그 제거
unique_tags.remove("국제")

# 두 태그 집합의 교집합 (공통 태그)
article1_tags = {"정치", "경제", "한국"}
article2_tags = {"경제", "금융", "한국"}
common = article1_tags & article2_tags
print(common)  # {'경제', '한국'}

6. 데이터 구조 비교표

구조 변경 가능 순서 중복 사용 예
리스트 ✓ 가능 ✓ 있음 ✓ 허용 기사 제목 목록, 취재원 명단
튜플 ✗ 불가 ✓ 있음 ✓ 허용 좌표, 고정 메타데이터
딕셔너리 ✓ 가능 ✓ 있음* ✗ 불가(키) 기사 정보, 설정값
집합 ✓ 가능 ✗ 없음 ✗ 불가 고유 태그, 중복 제거

* Python 3.7+에서는 딕셔너리도 삽입 순서를 유지합니다.

퀴즈: 어떤 데이터 구조를 쓸까?

Q1: 한 기사의 여러 정보(제목, 기자, 날짜, 조회수)를 함께 보관하려면?

답: 딕셔너리

각 정보를 키-값으로 저장하면, 제목이나 기자 이름으로 쉽게 접근할 수 있습니다.

article = {"제목": "...", "기자": "...", "날짜": "...", "조회수": 125000}

Q2: 100만 건의 기사에서 각 기사가 몇 번 언급되는지 세려면?

답: 딕셔너리

기사 제목을 키로, 언급 횟수를 값으로 저장하면 O(1)의 빠른 조회가 가능합니다.

article_count = {"기사제목1": 45, "기사제목2": 32, ...}

Q3: 한 기사의 모든 고유한 태그를 얻으려면?

답: 집합

tags = set(["정치", "정치", "경제", "경제"]) → {'정치', '경제'}

집합은 자동으로 중복을 제거하므로 고유 값만 남습니다.

주의: 리스트나 집합 같은 변경 가능한 객체를 딕셔너리의 키로 쓸 수 없습니다. 반드시 튜플이나 문자열 같은 불변 객체를 사용하세요.

같은 기사가 여러 번 언급될 때 카운트를 증가시키는 패턴은 자주 나옵니다:

Python 
# 기존 방법: KeyError 피하기 위해 매번 확인
article_count = {}
articles = ["정치", "정치", "경제", "정치"]

for article in articles:
    if article in article_count:
        article_count[article] += 1
    else:
        article_count[article] = 1

# 더 나은 방법: defaultdict 사용
from collections import defaultdict

article_count = defaultdict(int)  # 기본값은 0
for article in articles:
    article_count[article] += 1

print(dict(article_count))  # {'정치': 3, '경제': 1}

개요

알고리즘은 단순히 프로그래밍 개념이 아닙니다. 뉴스룸에서 데이터를 다루는 모든 작업에 알고리즘적 사고가 필요합니다.

1. 알고리즘이란?

알고리즘은 주어진 문제를 단계적으로 해결하는 절차입니다. 요리 레시피와 비슷합니다:

요리 레시피

  1. 쌀 2컵을 헹굼
  2. 물 2.5컵을 넣음
  3. 밥솥의 "취사" 버튼을 누름
  4. 30분 후 완성

기사 분류 알고리즘

  1. 모든 기사를 읽음
  2. 각 기사의 키워드를 확인
  3. 키워드에 따라 카테고리 분류
  4. 분류된 기사 저장
핵심: 알고리즘의 품질은 입력 데이터의 크기가 커질수록 더 중요해집니다. 100개 기사와 100만 개 기사에서 차이가 생깁니다.

2. 뉴스룸에서의 알고리즘적 사고

사례 1: 기사 분류 자동화

선거 결과 뉴스가 수천 건 들어올 때, 어떻게 자동으로 분류할까요?

Python 
# 기사 분류 알고리즘 (간단한 버전)
def classify_article(title):
    """기사 제목의 키워드로 카테고리 분류"""
    keywords = {
        "정치": ["국회", "선거", "정부", "여당", "야당"],
        "경제": ["삼성", "주식", "금리", "실적"],
        "환경": ["미세먼지", "날씨", "온난화"]
    }

    for category, kwords in keywords.items():
        for kword in kwords:
            if kword in title:
                return category

    return "기타"

# 사용 예
articles = [
    "국회, 신청사 이전 결정",
    "삼성전자 4분기 실적 발표",
    "내일 날씨 맑음"
]

for title in articles:
    category = classify_article(title)
    print(f"{title} → {category}")

3. 시간 복잡도 (Big-O): 알고리즘의 효율성

데이터 크기가 커질수록 알고리즘의 속도 차이가 극적으로 나타납니다. 이를 측정하는 것이 시간 복잡도(Big-O)입니다.

시간 복잡도 종류

표기 이름 설명 예시
O(1) 상수 시간 데이터 크기와 무관하게 항상 같은 시간 딕셔너리로 값 찾기
O(n) 선형 시간 데이터가 n배 증가하면 시간도 n배 증가 리스트 순회, 최댓값 찾기
O(n²) 이차 시간 데이터가 n배 증가하면 시간은 n²배 증가 이중 루프, 버블 정렬
O(log n) 로그 시간 데이터가 두 배 증가할 때 시간이 조금만 증가 이진 검색

실제 예: 100만 건 기사에서 검색할 때 차이

Python 
# O(1): 딕셔너리 조회 (매우 빠름)
article_dict = {"제목1": 100, "제목2": 200}
views = article_dict["제목1"]  # 즉시 반환

# O(n): 리스트 순회 (느림)
articles = ["제목1", "제목2", ...]  # 100만 개
target = "제목1000000"

found = False
for article in articles:
    if article == target:  # 평균 50만 번의 비교 필요
        found = True
        break

# O(n²): 이중 루프 (매우 느림)
# 100만 × 100만 = 1조 번의 비교 필요!
for i in articles:
    for j in articles:
        if i == j:
            pass

4. 정렬 알고리즘 비교

기사를 조회수 순서로 정렬하려면? 버블 정렬과 Python의 내장 정렬 함수의 차이를 봅시다.

Python 
# ❌ 버블 정렬: O(n²) - 느림
def bubble_sort(articles):
    n = len(articles)
    for i in range(n):
        for j in range(0, n-i-1):
            if articles[j]["조회수"] > articles[j+1]["조회수"]:
                articles[j], articles[j+1] = articles[j+1], articles[j]
    return articles

# ✓ Python 내장 sorted: O(n log n) - 빠름
articles = [
    {"제목": "기사1", "조회수": 100},
    {"제목": "기사2", "조회수": 350},
    {"제목": "기사3", "조회수": 250}
]

# 조회수 기준으로 내림차순 정렬

    articles,

# 두 문자열의 최장 공통 부분 길이
def lcs_length(text1, text2):
    m, n = len(text1), len(text2)

    # DP 테이블
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])

    return dp[m][n]

머신러닝이란?

머신러닝은 데이터를 학습하여 패턴을 찾고 예측하는 기술입니다. 전통적인 프로그래밍과 머신러닝의 차이를 이해하는 것이 중요합니다.

전통 프로그래밍 vs 머신러닝

전통 프로그래밍 머신러닝
규칙을 직접 코딩 → 결과 데이터 + 결과 → 규칙을 자동으로 학습
명시적인 if/else 로직 데이터 패턴에서 규칙 추출
변화에 따라 수동으로 코드 수정 새 데이터로 재학습하면 자동 개선
뉴스룸 예시: 스팸 댓글을 필터링하는 시스템을 만든다고 가정해봅시다.
  • 전통: "욕설 1000개 단어 목록 만들기" → 새로운 욕설이 나오면 계속 추가해야 함
  • ML: "스팸 10만 개, 정상 10만 개의 댓글 데이터 학습" → 자동으로 새로운 패턴 감지

머신러닝의 세 가지 유형

1. 지도학습 (Supervised Learning)

라벨이 있는 데이터를 사용하여 학습합니다.

  • 회귀(Regression): 연속된 숫자 예측 (예: 집값 예측, 주식 가격)
  • 분류(Classification): 범주 예측 (예: 이 기사는 정치/경제/스포츠?)

뉴스룸 활용: 신문사에 쏟아지는 기사 제안을 자동으로 분류 (속보/일반/광고)

2. 비지도학습 (Unsupervised Learning)

라벨이 없는 데이터에서 숨겨진 패턴을 찾습니다.

  • 클러스터링(Clustering): 비슷한 데이터끼리 묶기 (예: 비슷한 뉴스 기사 그룹화)
  • 차원 축소(Dimensionality Reduction): 복잡한 데이터를 단순화

뉴스룸 활용: 소셜미디어에서 떠오르는 주제들을 자동 감지

3. 강화학습 (Reinforcement Learning)

환경과의 상호작용을 통해 보상을 최대화하도록 학습합니다.

  • 시행착오를 통해 최적의 행동 학습
  • 게임 AI, 자율주행 등에 활용

뉴스룸 활용: 독자 선호도에 맞게 기사 추천 알고리즘 최적화

주의: 대부분의 저널리즘 분석에서는 지도학습을 사용합니다. 이 강의에서도 지도학습에 중점을 둡니다.

훈련 데이터와 테스트 데이터 분리

모델의 실제 성능을 정확히 측정하려면 훈련 데이터(Training Set)과 테스트 데이터(Test Set)를 분리해야 합니다.

왜 데이터를 분리할까?

만약 훈련에 사용한 데이터로 성능을 평가하면, 모델이 "이미 본 문제"에서는 높은 점수를 받지만, "새로운 문제"에서는 형편없는 성능을 보일 수 있습니다. 이것을 과적합(Overfitting)이라고 부릅니다.

비유: 시험 문제를 답만 외운 학생은 같은 문제는 맞히지만, 유사 문제는 풀 수 없습니다.

Python 
from sklearn.model_selection import train_test_split
import pandas as pd

# 한국 기사 데이터 로드
df = pd.read_csv('korean_articles.csv')

# X: 특성(features), y: 타겟(target)
X = df[['word_count', 'has_image', 'published_hour']]
y = df['category']  # 뉴스 카테고리

# 80:20으로 데이터 분리
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,  # 20%를 테스트로
    random_state=42  # 재현성을 위해 시드 설정
)

회귀 분석이란?

회귀 분석은 두 변수 사이의 관계를 찾아 하나의 변수로 다른 변수를 예측하는 기법입니다. 예를 들어, "독자 체류 시간"이 "기사 조회수"에 어떤 영향을 미치는지를 분석할 수 있습니다.

뉴스룸에서의 활용

  • 건강: 미세먼지 농도 ↔ 병원 응급실 방문 수
  • 경제: 광고 예산 ↔ 기사 클릭 수
  • 정치: 의원의 SNS 팔로워 ↔ 언론 보도 횟수
  • 부동산: 강남역까지의 거리 ↔ 평수당 아파트 가격

단순 선형 회귀 (Simple Linear Regression)

기본 개념

가장 간단한 형태로, X 변수 1개로 Y를 예측합니다.

Y = a + b × X

  • Y: 예측하려는 변수 (예: 기사 클릭 수)
  • X: 설명 변수 (예: 제목의 길이)
  • a: 절편 (X=0일 때의 Y값)
  • b: 기울기 (X가 1 증가할 때 Y의 변화)
Python 
import pandas as pd
import statsmodels.formula.api as smf

# 한국 도시별 데이터
df = pd.DataFrame({
    'city': ['서울', '부산', '대구', '인천', '광주'],
    'population_density': [10000, 4700, 2900, 2700, 1600],
    'grdp_per_capita': [45000, 38000, 33000, 35000, 28000]
})

# 회귀 모델 생성
model = smf.ols(formula='grdp_per_capita ~ population_density', data=df).fit()

# 결과 출력
print(model.summary())

결과 해석: R², p-value, 계수

회귀 결과에서 기자가 알아야 할 주요 지표들:

지표 의미 좋은 값
R² 모델이 데이터의 몇 %를 설명하는가?
(0~1, 클수록 좋음)		 0.7 이상
p-value 이 관계가 우연일 확률
(작을수록 신뢰도 높음)		 0.05 미만
Coefficient X가 1 증가할 때 Y의 변화량 비즈니스 의미 파악

기자 관점에서의 해석:

  • p-value < 0.05: "통계적으로 유의미하다" → 이 관계가 우연이 아니다
  • R² = 0.85: "인구밀도가 지역 경제 격차의 85%를 설명한다" → 강한 관계
  • Coefficient = 2.5: "인구밀도가 1000명/㎢ 증가하면, 1인당 GDP는 약 2500만원 증가한다"
중요: "상관관계 ≠ 인과관계"
두 변수가 함께 움직인다고 해서, 하나가 다른 하나를 원인으로 하는 것은 아닙니다. 예를 들어, "아이스크림 판매량"과 "익사 사건 수"는 상관관계가 있지만 (둘 다 여름에 증가), 인과관계는 없습니다 (실제 원인은 "날씨").

다중 선형 회귀 (Multiple Linear Regression)

여러 변수로 예측하기

현실 세계에서는 결과에 영향을 미치는 요소가 여러 개입니다.

Y = a + b₁×X₁ + b₂×X₂ + ... + bₙ×Xₙ

예: 서울 아파트 가격 = f(면적, 층수, 역세권여부, 건축연도, 반려동물 허용 여부)

Python 
# 서울 아파트 데이터 예시
apartment_data = pd.DataFrame({
    'price': [450, 380, 520, 290],  # 만원
    'area': [84, 59, 102, 49],  # 평수
    'floor': [15, 5, 25, 3],  # 층
    'near_station': [1, 0, 1, 0]  # 역세권 여부 (1=예, 0=아니오)
})

# 다중 회귀
model = smf.ols(
    formula='price ~ area + floor + near_station',
    data=apartment_data
).fit()

print(model.summary())
print(f"\nR² (설명력): {model.rsquared:.3f}")

다중공선성 (Multicollinearity) 주의

설명 변수들이 서로 높은 상관관계를 가지면, 모델의 신뢰도가 떨어집니다.

예: 아파트의 "건축연도"와 "경과년수"는 거의 같은 정보를 제공하므로, 두 변수를 함께 사용하면 불안정해집니다.

확인 방법: VIF (분산팽창계수) 계산

  • VIF < 5: 문제 없음
  • VIF > 10: 다중공선성 의심

로지스틱 회귀 (Logistic Regression)

기본 원리

로지스틱 회귀는 이름은 "회귀"이지만, 실제로는 분류 알고리즘입니다.

핵심 아이디어: 선형 회귀의 결과를 "확률"로 변환합니다.

시그모이드 함수 (Sigmoid Function)

P(Y=1) = 1 / (1 + e-z)

여기서 z = a + b×X

이 함수는 어떤 입력값이든 0과 1 사이의 확률값으로 변환합니다.

결과 해석:

  • P(Y=1) = 0.7 → 70% 확률로 이 뉴스는 가짜뉴스
  • P(Y=1) = 0.3 → 30% 확률로 이 뉴스는 가짜뉴스 (= 70% 확률로 진짜뉴스)

임계값 (Threshold) 설정

일반적으로 P(Y=1) ≥ 0.5이면 "Yes", 아니면 "No"로 분류합니다.

하지만 상황에 따라 임계값을 조정할 수 있습니다.

  • 임계값 = 0.3: 더 민감하게 "Yes"로 분류 (재현율 ↑, 정밀도 ↓)
  • 임계값 = 0.5: 기본값
  • 임계값 = 0.7: 더 보수적으로 "Yes"로 분류 (정밀도 ↑, 재현율 ↓)

모델 성능 평가

혼동행렬 (Confusion Matrix)

분류 모델의 성능을 보여주는 2×2 표입니다.

예측값
예측: Yes 예측: No
실제값 실제: Yes TP (참 양성) FN (거짓 음성)
실제: No FP (거짓 양성) TN (참 음성)
  • TP (True Positive): 가짜뉴스를 가짝뉴스라고 맞게 분류
  • FN (False Negative): 가짜뉴스를 진짜뉴스라고 잘못 분류 (위험!)
  • FP (False Positive): 진짜뉴스를 가짜뉴스라고 잘못 분류
  • TN (True Negative): 진짜뉴스를 진짜뉴스라고 맞게 분류

핵심 성능 지표

지표 공식 의미
정확도 (Accuracy) (TP + TN) / (TP + TN + FP + FN) 전체 예측 중 맞은 비율
전체적인 성능
정밀도 (Precision) TP / (TP + FP) "예"라고 예측한 것 중 실제로 맞은 비율
거짓 양성을 줄이고 싶을 때
재현율 (Recall) TP / (TP + FN) 실제 "예"인 것을 제대로 찾은 비율
놓친 것을 최소화하고 싶을 때
F1-Score 2 × (Precision × Recall) / (Precision + Recall) 정밀도와 재현율의 조화평균
균형잡힌 평가

뉴스룸 맥락: 왜 지표 선택이 중요한가?

가짜뉴스 탐지 시스템:

  • 높은 재현율이 필요: 가짜뉴스를 절대 놓치면 안 됨 (FN이 위험)
  • 낮은 정밀도 허용: 진짜뉴스를 실수로 걸러내는 것은 사람이 검수

스팸 댓글 필터:

  • 높은 정밀도가 필요: 정상 댓글을 스팸으로 지우면 안 됨 (FP가 위험)
  • 낮은 재현율 허용: 약간의 스팸이 통과해도 사용자가 피할 수 있음
Python 
# 성능 평가
cm = confusion_matrix(y_test, y_pred)
print("혼동행렬:")
print(cm)

# 자세한 분류 리포트
report = classification_report(y_test, y_pred,
                               target_names=['부결', '통과'])
print(report)

# 개별 지표 계산
tn, fp, fn, tp = cm.ravel()
accuracy = (tp + tn) / (tp + tn + fp + fn)
precision = tp / (tp + fp)
recall = tp / (tp + fn)
f1 = 2 * (precision * recall) / (precision + recall)

print(f"정확도: {accuracy:.2%}")
print(f"정밀도: {precision:.2%}")
print(f"재현율: {recall:.2%}")
print(f"F1-Score: {f1:.3f}")

의사결정나무 (Decision Trees)

중급-고급 50분

의사결정나무란?

의사결정나무(Decision Tree)는 마치 "스무고개" 게임처럼 일련의 질문을 통해 데이터를 분류하는 머신러닝 모델입니다. 각 노드에서 조건을 확인하고 가지를 따라가며 최종 결정에 도달합니다.

스무고개 비유: "나이가 30 이상인가? → 구독 기간이 1년 이상인가? → 방문 빈도가 주 3회 이상인가?" 이런 식으로 질문하며 최종 판단에 도달하는 과정입니다.

뉴스룸 실전 예시: 독자 이탈 예측

  • 독자 나이 > 30세? ✓
  • → 구독 기간 > 1년? ✗
  • → 최근 3개월 방문 횟수 > 5회? ✓
  • → 예측: 이탈 가능성 낮음

특성 엔지니어링 (Feature Engineering)

고급 45분

특성 엔지니어링이란?

특성 엔지니어링(Feature Engineering)은 원본 데이터에서 기계학습 모델에 입력할 수 있는 의미 있는 특성을 만드는 과정입니다.

GIGO 원칙 (Garbage In, Garbage Out): 아무리 좋은 모델도 나쁜 특성으로는 좋은 결과를 기대할 수 없습니다. 좋은 특성이 좋은 모델을 만듭니다!

data_normalized = scaler_minmax.fit_transform(data_raw)

# 표준화 (평균 0, 표준편차 1)
scaler_standard = StandardScaler()
data_standardized = scaler_standard.fit_transform(data_raw)

print("정규화:")
print(data_normalized)
print("\n표준화:")
print(data_standardized)
정규화 vs 표준화:
  • 정규화: 데이터를 [0, 1] 범위로 변환. 특성의 범위가 다를 때 사용 (예: KNN, K-means)
  • 표준화: 데이터를 평균 0, 표준편차 1로 변환. 정규분포 가정 모델에 유용 (예: 로지스틱 회귀, SVM)

모델 검증과 교차검증 (Model Validation & Cross-Validation)

고급 45분

왜 모델 검증이 중요한가?

뉴스룸에서 ML 모델을 사용한다면, 그 모델이 얼마나 신뢰할 수 있는지 검증해야 합니다. 잘못된 예측으로 팩트체크 없이 보도하면 큰 문제가 발생합니다.

실제 사례: "이 모델은 훈련 데이터에서 99% 정확도를 보였습니다"라고 해도, 새로운 데이터에서 50% 정확도만 나올 수 있습니다. 이것이 과적합(overfitting)입니다!

Holdout 검증

데이터를 훈련(train)과 테스트(test) 세트로 나누어 검증합니다. 하지만 데이터가 적을 때는 한계가 있습니다.

Python 
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

# 데이터 분할 (70% 훈련, 30% 테스트)
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.3,
    random_state=42
)

# 모델 학습
model = LogisticRegression(random_state=42)
model.fit(X_train, y_train)

# 검증
train_score = model.score(X_train, y_train)
test_score = model.score(X_test, y_test)

print(f"훈련 정확도: {train_score:.3f}")
print(f"테스트 정확도: {test_score:.3f}")
print(f"차이: {train_score - test_score:.3f} (클수록 과적합)")
Holdout의 한계: 데이터가 100개 미만이면 30개 테스트 세트는 너무 작아서 신뢰성이 떨어집니다. 이때는 교차검증을 사용하세요.

K-fold 교차검증

데이터를 K개로 나누어, 각각을 돌아가며 테스트 세트로 사용합니다. 더 안정적인 성능 평가가 가능합니다.

Python 
from sklearn.model_selection import cross_val_score

# 5-fold 교차검증

scores = cross_val_score(
    model,
    X, y,
    cv=5  # 5개 폴드
)

print(f"각 폴드 정확도: {scores}")
print(f"평균 정확도: {scores.mean():.3f}")
print(f"표준편차: {scores.std():.3f}")

해석:

  • 5개 폴드가 각각: [0.92, 0.88, 0.90, 0.89, 0.91]
  • 평균: 0.900 (모델의 예상 성능)
  • 표준편차: 0.014 (성능의 안정성 - 작을수록 좋음)

나이브 베이즈란?

나이브 베이즈(Naive Bayes)는 확률 기반 분류 알고리즘으로, 특히 텍스트 분류에 매우 효과적입니다. "나이브(naive)"라는 이름은 각 특성이 서로 독립적이라는 단순한(naive) 가정 때문입니다.

실생활 예시: 스팸 필터가 이메일을 스팸인지 판단할 때, "무료"라는 단어가 있으면 스팸일 확률이 높다고 학습합니다. 나이브 베이즈는 이런 식으로 단어들의 확률을 계산합니다.

베이즈 정리와 직관적 이해

기사 제목에 "속보"라는 단어가 있으면 실제 속보(breaking news)일 확률은?

  • 사전확률 P(속보): 전체 기사 중 속보의 비율 (예: 5%)
  • 우도 P("속보" | 속보): 실제 속보 기사 중 "속보"라는 단어가 있을 확률 (예: 90%)
  • 증거 P("속보"): 전체 기사 중 "속보"라는 단어가 있을 확률
  • 사후확률: "속보"라는 단어가 있을 때 실제 속보일 확률
Python 
# 베이즈 정리 예시: 속보 감지
# P(속보 | "속보" 단어 있음) = P("속보" | 속보) * P(속보) / P("속보")

P_breaking = 0.05          # 사전확률: 5%는 속보
P_word_given_breaking = 0.9  # 우도: 속보의 90%가 "속보" 단어 포함
P_word = 0.15           # 증거: 전체 기사의 15%가 "속보" 단어 포함

# 사후확률 계산
P_breaking_given_word = (P_word_given_breaking * P_breaking) / P_word

print(f"'속보' 단어가 있으면 실제 속보일 확률: {P_breaking_given_word:.1%}")
